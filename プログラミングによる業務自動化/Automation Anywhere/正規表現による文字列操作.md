# Automation 360 正規表現による文字列操作

## 概要

Automation Anywhere Automation 360 では、組み込みの **String パッケージ**と Bot Store からインストールできる **Regex Tools パッケージ**を使って正規表現による文字列操作を行う。

---

## 1. 正規表現が使えるパッケージ・コマンド

### 1.1 String パッケージ（組み込み）

Automation 360 に標準で含まれるパッケージ。以下のアクションで正規表現を利用できる。

| アクション名 | 用途 | 正規表現サポート |
|-------------|------|----------------|
| Find（検索） | 文字列内で部分文字列を検索し、最初に出現する位置（インデックス）を返す | あり |
| Replace（置換） | 文字列内のテキストを別のテキストで置き換える | あり |
| Extract text（テキストを抽出） | 指定した文字列の前後を区切りとして範囲テキストを抽出する | なし（区切り文字方式） |

**String パッケージの全アクション一覧**（正規表現非対応含む）

- Assign（代入）
- Compare（比較）
- Evaluate variable（変数を評価）
- Extract text（テキストを抽出）
- Find（検索）
- Generate random string（ランダム文字列を生成）
- Import string from text file（テキストファイルから読み込み）
- Length（長さ取得）
- Lowercase（小文字変換）
- Replace（置換）
- Substring（部分文字列取得）
- To number（数値に変換）
- Uppercase（大文字変換）

#### Find アクション（String パッケージ）

**入力パラメータ**

| パラメータ | 内容 |
|-----------|------|
| Source string（元の文字列） | 検索対象の文字列 |
| Search string（検索文字列） | 検索する文字列または正規表現パターン |
| Search string is（検索文字列の種類） | `A regular expression`（正規表現）または `Not a regular expression`（通常文字列）を選択 |
| Start from（開始位置） | 検索開始位置（省略可、デフォルト: 1） |

**出力**

マッチした最初の位置のインデックス（Number 型変数）を返す。マッチしない場合は 0 を返す。

> **注意**: If 条件の「Regex Match is Found」が期待通りに動かない場合、String > Find アクションを使い、出力が 1（マッチあり）か 0（マッチなし）かで分岐する方法が推奨されている。

#### Replace アクション（String パッケージ）

**入力パラメータ**

| パラメータ | 内容 |
|-----------|------|
| Source string（元の文字列） | 処理対象の文字列 |
| Search string（検索文字列） | 置換対象のテキストまたは正規表現パターン |
| Replacement string（置換文字列） | 置換後のテキスト |
| Regular expression（正規表現） | チェックボックス。有効にすると正規表現として解釈 |
| Match case（大文字小文字を区別） | チェックボックス。有効にすると大文字小文字を区別 |
| Start from（開始位置） | 置換開始位置（省略可、デフォルト: 1） |
| Count（回数） | 置換回数。空または -1 で最後まですべて置換 |

**出力**

置換後の文字列（String 型変数またはクリップボード）

---

### 1.2 Regex Tools パッケージ（Bot Store からインストール）

Bot Store（Agentic App Store）から無償でインストールできるカスタムパッケージ。正規表現専用のアクションを 4 つ提供する。

- Bot Store URL: https://botstore.automationanywhere.com/bot/a2019-regular-expression-tools-package

**導入方法**: Control Room の Bot Store からパッケージをインポートし、Bot Editor の Action Palette から使用する。

#### アクション一覧

| アクション名 | 機能 |
|-------------|------|
| Extract All Regex Matches | 指定パターンに一致するすべての文字列をリストとして抽出 |
| Extract a Regex Match | 指定パターンに一致する N 番目の文字列を抽出 |
| Get Number of Matches | 指定パターンが出現する回数を取得 |
| Replace based on Regex | 指定パターンに一致する部分を別の文字列で置換 |

#### Extract All Regex Matches

- **入力**: ソース文字列、正規表現パターン
- **出力**: マッチした文字列のリスト（List 型変数）
- マッチが 0 件の場合は空リストを返す
- リストに対して For Each ループでマッチ結果を順に処理できる

#### Extract a Regex Match

- **入力**: ソース文字列、正規表現パターン、取得するマッチの番号（N）
- **出力**: N 番目にマッチした文字列（String 型変数）
- 最初のマッチは N=1（または N=0、実装による）

#### Get Number of Matches

- **入力**: ソース文字列、正規表現パターン
- **出力**: マッチ回数（Number 型変数）
- マッチなしの場合は 0 を返す
- 文字列のバリデーション（数字のみか、メールアドレス形式か等）の用途に使える

#### Replace based on Regex

- **入力**: ソース文字列、正規表現パターン、置換文字列
- **出力**: 置換後の文字列（String 型変数）

---

### 1.3 If モジュールでの正規表現マッチ確認

`If` アクションのコンディションで「Regex Tool > Regex Match is Found」を選択することで、正規表現パターンにマッチするかどうかを条件分岐に使える。

ただし、コミュニティでの報告によるとこの条件が期待通りに動作しないケースがある。その場合は以下の回避策を使用する。

**回避策**: Regex Tools > Get Number of Matches を使い、マッチ数が 1 以上かどうかを条件として評価する。

## 2. 正規表現の文法

### 2.1 正規表現エンジン

Automation 360 の正規表現エンジンは **Java の `java.util.regex`** をベースとしている。パッケージの SDK が Java で実装されており、内部で `java.util.regex.Pattern` / `java.util.regex.Matcher` クラスを使用している。

Java 正規表現エンジンは NFA（Non-deterministic Finite Automaton）型で、.NET、Perl、Python と同系統の構文を採用している。

### 2.2 利用可能なフラグ

公式ドキュメントにはフラグの明示的な一覧は記載されていない。コミュニティの情報によれば以下の方法でフラグを設定できる。

| フラグ | 埋め込みフラグ記法 | 効果 |
|--------|------------------|------|
| 大文字小文字を区別しない | `(?i)` をパターン先頭に付与 | 例: `(?i)hello` で Hello/HELLO にもマッチ |
| 複数行モード | `(?m)` をパターン先頭に付与 | `^` と `$` が各行の先頭・末尾にマッチ |
| DOTALLモード | `(?s)` をパターン先頭に付与 | `.` が改行文字にもマッチ |

Replace アクションには「Match case（大文字小文字を区別）」チェックボックスがあり、正規表現のフラグを使わずに大文字小文字の区別を制御できる。

### 2.3 Automation 360 固有のエスケープ規則

| 注意点 | 内容 |
|--------|------|
| `$` の二重記述 | `$` を正規表現パターンに含める場合、`$$` と二重に書く。Automation 360 の変数補間と競合するため |
| バックスラッシュ | 通常通り `\d`、`\w` 等を使用可能 |
| キャプチャグループの参照 | 置換文字列でグループを参照する場合、`$$1`（`$1` のエスケープ）と記述する |

## 3. 実用的な使用例

### 3.1 メールアドレスの抽出

**使用アクション**: String > Find または Regex Tools > Extract All Regex Matches

```
パターン: \b[A-Z0-9._%-]+@[A-Z0-9.-]+\.[A-Z]{2,4}\b

大文字小文字を区別しない場合:
(?i)\b[A-Z0-9._%-]+@[A-Z0-9.-]+\.[A-Z]{2,4}\b
```

複数のメールアドレスを抽出する場合は「Extract All Regex Matches」を使い、リストで受け取って For Each ループで処理する。

### 3.2 日付パターンの抽出（YYYY/MM/DD）

**使用アクション**: Regex Tools > Extract a Regex Match

```
パターン: \d{4}[年\-\/]\d{1,2}[月\-\/]\d{1,2}日?
```

西暦日付（スラッシュ・ハイフン・漢字区切り対応）を抽出できる。

### 3.3 文字列の検証（数字のみかチェック）

**使用アクション**: Regex Tools > Get Number of Matches

```
パターン: ^\d+$
```

マッチ数が 1 なら数字のみ、0 なら数字以外を含む。

**または** String > Find アクション

```
パターン: ^\d+$
検索文字列の種類: A regular expression
```

出力が 1 ならマッチ（数字のみ）、0 ならマッチなし（数字以外を含む）。

### 3.4 正規表現による文字列置換

**使用アクション**: String > Replace または Regex Tools > Replace based on Regex

**例1: 日付形式の変換（YYYY/MM/DD → YYYYMMDD）**

```
元の文字列: 2024/06/17
パターン: (\d{4})/(\d{2})/(\d{2})
置換文字列: $$1$$2$$3
結果: 20240617
```

**例2: 数字のみ削除してアルファベットを抽出**

```
パターン: \d+
置換文字列: （空）
```

**例3: ファイルパスからファイル名のみ抽出**

```
パターン: ^.*\\
置換文字列: （空）
例: C:\folder\test.txt → test.txt
```

**例4: 郵便番号の抽出（ハイフンあり・なし対応）**

```
元の文字列（全体にマッチするパターン）: .*(\d{3}-?\d{4}).*
置換文字列: $$1
```

### 3.5 複数マッチをリストで取得してループ処理

**使用アクション**: Regex Tools > Extract All Regex Matches

手順:
1. Regex Tools > Extract All Regex Matches を実行
   - Source string: 処理対象のテキスト（例: Webスクレイピング結果）
   - Pattern: 抽出したいパターン（例: `\b\d{3}-\d{4}\b` で電話番号）
   - 出力: List 型変数（例: `listMatches`）
2. Loop > For Each アクションで `listMatches` をイテレート
3. ループ内で各マッチ（String 型変数）を処理

## 4. 注意事項・ハマりポイント

### 4.1 `$` の二重エスケープ

Automation 360 では `$` が変数参照の記号として機能するため、正規表現パターンや置換文字列に `$` をリテラルとして含める場合は `$$` と書く必要がある。

```
例: 行末にマッチ → $$（Automation 360 内では $$ と入力）
キャプチャグループ参照 → $$1（通常の $1 の代わりに）
```

### 4.2 If 条件の「Regular expression」オプションの制限

If アクションの条件で「Regex Match is Found」を直接使うとマッチしないケースがある（コミュニティ報告）。`^8` のような行頭マッチが機能しないなどの問題が確認されている。

**代替手段**:
- String > Find アクションを使い、出力（1/0）を条件変数に格納して判断する
- Regex Tools > Get Number of Matches を使い、マッチ数 > 0 を条件にする

### 4.3 高度な正規表現置換の制限

String > Replace アクションの正規表現置換では、`\U$1\L$2` のような大文字小文字変換の置換構文（.NET 独自記法）は動作しない。Java の `String.replaceAll()` ベースのため、バックリファレンス（`$$1`）のみサポートしている。

### 4.4 正規表現エンジンの動作確認ツール

Automation 360 は Java の `java.util.regex` を使用しているため、パターンの検証には以下のツールが適している。

- [regex101.com](https://regex101.com/) でフレーバーを「Java 8」に設定して確認
- Java の正規表現構文リファレンスを参照

### 4.5 Regex Tools パッケージはインストールが必要

Regex Tools パッケージは標準では含まれておらず、Bot Store からインストールが必要。Control Room の管理者権限でパッケージをインポートし、Packages リポジトリで有効化する必要がある。

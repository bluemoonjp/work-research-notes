# 言語別解説: 業務システムからExcelダウンロードして集計

## C#

### record と class の使い分け

| | `record SalesRecord` | `class CategorySummary` |
|---|---|---|
| 表すもの | Excel から読み込んだ元データ（実体） | LINQ が計算した集計結果（プログラムの産物） |
| 変更可否 | 変更不可（不変） | 変更不可（`init` で制御） |
| 等値比較 | 値が同じなら同じとみなせる | 参照（インスタンス）で比較 |
| 向いている場面 | 外部から取り込んだデータ、DTO | 計算結果、サービスクラスの出力 |

`record` にするメリット:
- 読み込んだ後にデータが書き換わらないことをコードで表明できる
- `ToString()` が自動で `SalesRecord { OrderDate = 2024-06-01, ProductName = ... }` のように出力されデバッグしやすい

### LINQ の利点

`SalesAggregator.Aggregate` の中で対比コメントを書いているが、要点をまとめると:

**可読性**
- `GroupBy → Select → OrderByDescending` と操作が英語で連なるため、何をしているか一目で分かる
- ループ変数 (`i`, `j`) や中間 Dictionary が不要になりコードが短くなる

**複合集計の効率**
- 1 回の `GroupBy` で合計金額・合計数量・件数を同時に取れる
- ループで書くと Dictionary を 3 つ管理する必要がある

**拡張しやすい**
- `Where(r => r.OrderDate >= "2024-06-01")` を先頭に追加するだけで絞り込みを加えられる
- ループ版では if 文を追加する位置を探す必要がある

**注意点**
- `ToList()` を忘れると遅延評価のままになり、後続で何度もループが走る場合がある
- 複雑な集計を 1 行に詰め込みすぎると逆に読みにくくなる

### async / await

Playwright は非同期処理（async/await）が前提のライブラリ。

- `async Task MethodName()` — 非同期メソッドの宣言。`await` を使うために必要
- `await someTask` — そのタスクが完了するまで待ってから次の行へ進む
- エントリポイントも `static async Task Main` にすることで `await` が使える

同期処理（通常の `void` メソッド）との違いをイメージで言うと:
- 同期: 「コーヒーができるまでカウンターの前で立って待つ」
- 非同期: 「番号札をもらって席で待ち、できたら呼ばれる」

ブラウザ操作はネットワーク遅延やページ遷移待ちが発生するため、非同期が適している。

### ClosedXML を選んだ理由

| ライブラリ | ライセンス | 特徴 |
|---|---|---|
| **ClosedXML** | MIT（商用無料） | 読み書き両対応・Office 不要・API が直感的 |
| EPPlus 5+ | PolyForm（非商用のみ無料） | 高機能だが商用利用は有償 |
| COM Interop | — | Excel インストール必須・サーバー利用不可 |

このサンプルでは ClosedXML で読み込みと書き込みの両方を行う。
同じライブラリを使うことで API の学習コストが最小になる。

### Playwright を選んだ理由（Selenium との比較）

| | Playwright | Selenium |
|---|---|---|
| ダウンロード処理 | `RunAndWaitForDownloadAsync` で組み込み対応 | ダウンロード先フォルダを ChromeOptions で事前設定・ファイル出現を手動ポーリング |
| 待機処理 | 多くの操作に自動待機が組み込まれている | `WebDriverWait` で条件を明示的に書く必要がある |
| 非同期 | async/await ネイティブ | 基本は同期 API（非同期ラッパーは別途必要） |
| 初回セットアップ | `playwright install chromium` が必要 | ChromeDriver のバージョンを手動で合わせる必要がある |
| ブラウザ互換 | Chromium / Firefox / WebKit | Chrome / Firefox / Edge など |

業務システムのダウンロード自動化では、Playwright の組み込みダウンロードサポートが特に効果的。

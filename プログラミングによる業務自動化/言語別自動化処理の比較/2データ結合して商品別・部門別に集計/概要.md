# 2データを結合して商品別・部門別に集計

## 処理の目的

ログインが必要な社内 Web システムから **商品マスタ** と **売上データ** の 2 つの Excel をダウンロードする。
LINQ の Join で 2 データを結合し、商品別・部門別それぞれの集計シートを作成する。

## 想定シナリオ

- 社内 Web システムにログインし、先月の日付範囲で絞り込む
- 「商品マスタダウンロード」ボタンと「売上データダウンロード」ボタンをそれぞれクリックして 2 ファイル取得
- 商品コードをキーに 2 データを結合し、以下の集計を行う:
  - 商品ごとの売上金額・件数・販売個数・販売ケース数・端数（入り数も出力）
  - 部門カテゴリごとの客単価（売上合計 ÷ ユニークレシート数）

コード内の URL・セレクタ・ログイン情報はすべてダミー。実環境では定数を差し替える。

## 前提条件

### NuGet パッケージ

| パッケージ | 用途 | ライセンス |
|---|---|---|
| `Microsoft.Playwright` | ブラウザ操作 | Apache 2.0（商用無料） |
| `ClosedXML` | Excel 読み書き | MIT（商用無料） |

### 初回セットアップ

```
dotnet tool install --global Microsoft.Playwright.CLI
playwright install chromium
```

## 入力データのフォーマット

### 商品マスタ Excel（1シート・1行目ヘッダ）

| 列 | 項目 | 型 |
|---|---|---|
| A | 商品コード | 文字列（売上データとの結合キー） |
| B | 商品名 | 文字列 |
| C | 部門カテゴリ | 文字列 |
| D | 入り数 | 整数（1ケース何個入りか） |

### 売上データ Excel（1シート・1行目ヘッダ）

| 列 | 項目 | 型 |
|---|---|---|
| A | レシートID | 文字列（客単価の客数カウントに使用） |
| B | 売上日 | 文字列 |
| C | 商品コード | 文字列（結合キー） |
| D | 販売個数 | 整数 |
| E | 売上金額 | decimal |

## 注意事項

- Join は SQL の INNER JOIN と同じ動作をする。商品マスタに存在しない商品コードの売上行は結合後に除外される。
- 客単価はレシートID の重複を除いた件数（= 来客数）で割る。同一レシートに複数の部門の商品が含まれる場合、各部門に同じレシートIDがカウントされる。

## 入力 / 出力

| 区分 | 内容 |
|---|---|
| 入力 | コード内の定数（ログイン情報・URL） |
| ダウンロード | 商品マスタ Excel・売上データ Excel |
| 出力 | `集計結果.xlsx`（結合データ・商品別集計・部門別集計の 3 シート） |

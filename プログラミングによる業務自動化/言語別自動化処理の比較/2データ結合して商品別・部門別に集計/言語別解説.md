## C#

### DateTime の操作（前回サンプルから追加）

C# の `DateTime` は日付計算のメソッドが充実しており、月末の計算でも手動で日数を数える必要がない。

```csharp
var today            = DateTime.Now;                              // 今日の日時
var firstOfThisMonth = new DateTime(today.Year, today.Month, 1); // 今月1日
var lastMonthStart   = firstOfThisMonth.AddMonths(-1);           // 先月1日
var lastMonthEnd     = firstOfThisMonth.AddDays(-1);             // 先月末

// ToString で書式を指定して文字列に変換する
// "yyyy/MM/dd" → "2024/05/31" のような形式
// フォームの形式に合わせて "yyyy-MM-dd" や "MM/dd/yyyy" などに変更できる
var startStr = lastMonthStart.ToString("yyyy/MM/dd");
```

**うるう年・月ごとの日数差を自動で吸収する**のが大きなメリット。
たとえば「3月1日の AddDays(-1)」は自動で 2月28日（うるう年なら29日）になる。

### LINQ Join: 2つのリストの結合

`DataJoiner.Join` の中で対比コメントを掲載しているが、要点をまとめると:

**構文の対応関係（SQL との比較）**

| SQL | LINQ |
|---|---|
| `FROM sales s` | `salesRows.Join(` |
| `JOIN master m` | `productMasters,` |
| `ON s.code = m.code` | `s => s.ProductCode, p => p.ProductCode,` |
| `SELECT s.*, m.*` | `(s, p) => new JoinedSales(...)` |

**INNER JOIN の挙動**（注意点）
- 商品マスタに存在しない `ProductCode` の売上行は結合後に**消える**
- 「売上はあるが商品マスタにない」行を保持したい場合は `GroupJoin` + `SelectMany`（LEFT JOIN 相当）を使う

### 整数除算 `/` と剰余 `%`

`SalesAggregator.ByProduct` でケース数・端数を計算する部分:

```csharp
// C# では int / int の結果は int（小数点以下が切り捨て）
// これを利用してケース数を求める
Cases     = totalQty / unitsPerCase;  // 例: 25 / 6 = 4

// % は剰余演算子（割り算の余り）
// ケースに満たない端数を求める
Remainder = totalQty % unitsPerCase;  // 例: 25 % 6 = 1
```

`decimal` や `double` との混在に注意:
- `int / int` → `int`（切り捨て）
- `decimal / int` → `decimal`（精度を保持）
- ケース数・端数のように「個数の整数演算」では `int` の整数除算が意図通りに動く

### `Distinct().Count()` で重複除外カウント

`SalesAggregator.ByDepartment` の客単価計算:

```csharp
// Select でひとつのプロパティだけを取り出す → IEnumerable<string>
// Distinct で重複を除去する（同じレシートIDが複数行あっても 1 件と数える）
// Count で件数を確定させる
var customerCount = g.Select(r => r.ReceiptId).Distinct().Count();
```

`Distinct` を使わない場合、同一レシートの複数明細がそれぞれ 1 客とカウントされてしまう。

### タプル戻り値 `(T1, T2)`

`BusinessSystemDownloader.DownloadBothAsync` の戻り値に使用:

```csharp
// 戻り値の型に名前付きタプルを使う
public async Task<(string masterPath, string salesPath)> DownloadBothAsync(...)

// 呼び出し側は分解代入で受け取れる（見た目が直感的）
var (masterPath, salesPath) = await downloader.DownloadBothAsync(workDir);
```

`out` パラメータとの比較:

| | タプル戻り値 | out パラメータ |
|---|---|---|
| 書き方 | 戻り値として自然 | メソッド引数に `out` をつける |
| 非同期メソッド | 使える（`async Task<(T1, T2)>`） | 使えない（async メソッドに out 不可） |
| 向いている場面 | 2〜3個の関連する値を返す | 既存 API（`TryParse` など）との一貫性 |

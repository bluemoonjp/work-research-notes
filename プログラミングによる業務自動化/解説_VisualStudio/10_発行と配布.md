# 10: 発行と配布

## この記事で学ぶこと

- ビルド出力と発行（Publish）の違い
- フレームワーク依存と自己完結型の選択基準
- 発行プロファイルの作成手順
- 配布時の注意点

---

## ビルド出力と発行の違い

| 操作 | 結果 | 用途 |
|------|------|------|
| ビルド（F6） | `bin/Release/net8.0/` に出力 | 開発中の確認用 |
| 発行（Publish） | 指定フォルダに必要なファイルだけを出力 | 配布用 |

**ビルド出力をそのまま渡すと `obj/` フォルダなど不要なファイルが含まれる。** 配布するときは必ず発行機能を使う。

---

## フレームワーク依存 vs 自己完結型

### フレームワーク依存（Framework-Dependent）

実行には配布先 PC に .NET ランタイムがインストールされている必要がある。

- ファイルサイズ: 小さい（数百 KB 程度）
- 前提: 配布先 PC に同じバージョンの .NET ランタイムが必要

### 自己完結型（Self-Contained）

.NET ランタイムをアプリに同梱して配布する。配布先に .NET をインストール済みかどうかを気にしなくてよい。

- ファイルサイズ: 大きい（数十〜100 MB 程度）
- 前提: なし

**業務 PC に .NET ランタイムが入っているか不明な場合は自己完結型が安全。**

---

## 発行の手順

1. ソリューション エクスプローラーでプロジェクトを右クリック → **発行**
2. **発行先**: フォルダーを選択
3. 出力先フォルダを指定（例: `C:\publish\MyApp`）
4. **発行プロファイルの設定** で以下を選択:

| 設定項目 | 推奨 |
|----------|------|
| 構成 | Release |
| ターゲット フレームワーク | プロジェクトと同じバージョン |
| 配置モード | 自己完結型（配布先に .NET 不要にしたい場合） |
| ターゲット ランタイム | win-x64（64ビット Windows 向け） |
| ファイルの生成 | 単一ファイル（オプション）|

5. **発行** ボタンをクリック

---

## 単一ファイル出力

「単一ファイルとして発行」を有効にすると、複数の DLL が 1 つの `.exe` にまとまる。配布がシンプルになるが、起動が少し遅くなる場合がある。

---

## 発行後の確認

発行フォルダを別の PC（または VM）で実際に実行して動作確認する。自分の開発環境だけで確認していると、依存関係の漏れに気づかない場合がある。

---

## 配布時のチェックリスト

- [ ] Release ビルドの構成で発行しているか
- [ ] 配布先 PC の .NET ランタイム要件を確認したか（または自己完結型を選択しているか）
- [ ] 設定ファイル（appsettings.json など）や外部ファイルも一緒に含めているか
- [ ] 別環境で実際に動作確認したか

---

## よくある落とし穴

- **Debug ビルドで発行する**: ビルド構成が Debug のまま発行してしまうと、デバッグ情報が含まれ動作も遅くなる。必ず Release を選択する
- **設定ファイルを忘れる**: アプリが読み込む設定ファイルや Excel テンプレートなどは、発行ウィザードで「コピーしない」になっていると出力されない。プロジェクトのプロパティで各ファイルの「出力ディレクトリにコピー」を「常にコピーする」に設定する
- **ターゲット ランタイムの不一致**: 配布先が 32 ビット環境なのに win-x64 でビルドすると実行できない。配布先の環境に合わせて選択する

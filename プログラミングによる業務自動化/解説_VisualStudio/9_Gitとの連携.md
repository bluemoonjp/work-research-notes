# 9: Git との連携

## この記事で学ぶこと

- Visual Studio 内でできる Git 操作
- リポジトリの初期化とクローン
- コミット・ブランチ操作の基本フロー
- 外部ツール（CLI・GitHub Desktop）との使い分け

---

## Visual Studio 内の Git 機能

Visual Studio 2022 は Git 操作を統合する UI を標準搭載している。**表示 > Git 変更**（Ctrl+0, Ctrl+G）でパネルを開く。

主な操作はすべてこのパネルかメニューバーの **Git** メニューから行える。

---

## リポジトリの初期化

新しいプロジェクトを作成するとき、プロジェクト作成ダイアログで「Git リポジトリに配置する」チェックボックスをオンにすると自動で初期化される。

既存プロジェクトに後から Git を追加するには:

1. **Git > Git リポジトリの作成**
2. ローカルのみ（リモートなし）または GitHub に接続するかを選ぶ

---

## 基本的な変更〜コミットの流れ

```
コードを変更
  ↓
Git 変更パネルで変更ファイルを確認
  ↓
ステージング（「+」ボタンでファイルを選択）
  ↓
コミットメッセージを入力してコミット
  ↓
（必要に応じて）プッシュ
```

**「すべてコミット」ボタン**は変更ファイルをステージングとコミットを一括で行う。差分を確認せずにコミットすることになるため、レビュー習慣をつけるならファイルごとにステージングする方がよい。

---

## 差分の確認

変更ファイルをダブルクリックすると、変更前後の差分が左右分割で表示される。コミット前に何を変えたか確認するのに使う。

---

## ブランチ操作

Visual Studio の画面右下または **Git 変更パネル** でブランチ名が表示される。

| 操作 | 手順 |
|------|------|
| ブランチの作成 | ブランチ名をクリック → **新しいブランチ** |
| ブランチの切り替え | ブランチ名をクリック → 一覧から選択 |
| マージ | **Git > 管理 > ブランチ** でマージ先を選択 |

---

## 履歴の確認

**Git > ファイル履歴の表示** または **ソリューション エクスプローラー** でファイルを右クリック → **Git > 履歴の表示** で、そのファイルの変更履歴が表示される。

各コミットをクリックすると、そのコミットの差分を確認できる。

---

## 外部ツールとの使い分け

| ツール | 向いている場面 |
|--------|--------------|
| Visual Studio 内 Git UI | コーディング中に素早くコミット・差分確認 |
| Git CLI（コマンドライン） | 複雑な操作（rebase・cherry-pick・stash など） |
| GitHub Desktop | GUI で直感的に操作したい場合。リモートとの同期が分かりやすい |

Visual Studio の Git UI は基本操作には十分だが、`git rebase` や `git stash` などの高度な操作は CLI のほうが扱いやすい。両方を状況に応じて使い分けるのが実務では一般的。

---

## よくある落とし穴

- **`.gitignore` が設定されていない**: `bin/` や `obj/` フォルダがコミットされてしまう。Visual Studio が自動生成する `.gitignore` を使えば C# 向けの設定が入っている
- **コンフリクトが発生したとき**: Visual Studio の **コンフリクトの解決** UI が開く。「現在の変更を受け入れる」「着信した変更を受け入れる」「両方を受け入れる」から選ぶか、手動で編集する

# 1: 最初にした方がいい設定

## この記事で設定すること

- テーマとフォントを見やすく変える
- 行番号を表示する
- 行末文字とエンコーディングを確認する
- GitHub Copilot / IntelliCode の自動補完を無効化する
- その他の細かい設定

すべて **ツール > オプション** から変更できる。

---

## テーマとフォント

### カラーテーマ

**ツール > オプション > 環境 > 全般 > 配色テーマ**

| テーマ | 特徴 |
|--------|------|
| Dark | 暗背景。長時間作業向き |
| Light | 明背景。印刷物や発表向き |
| Blue | デフォルト。Light に近い |

### フォントとサイズ

**ツール > オプション > 環境 > フォントおよび色**

- 表示の設定対象: **テキスト エディター**
- フォント: Consolas（デフォルト）または BIZ UDゴシック・HackGen など
- サイズ: 12〜14pt が一般的

---

## 行番号の表示

**ツール > オプション > テキスト エディター > すべての言語 > 全般 > 行番号**

チェックを入れるだけ。エラーメッセージに「〇行目」と表示されたとき、すぐ確認できるようになる。

---

## 行末文字とエンコーディング

Visual Studio はデフォルトで CRLF（Windows 改行）を使用する。Git リポジトリを共有する場合は `.editorconfig` や `.gitattributes` で統一するのが望ましいが、個人開発では特に変更しなくてもよい。

エンコーディングは **ファイル > 名前を付けて保存 > 保存ボタン右の▼ > エンコード付きで保存** から確認・変更できる。業務で使う場合は UTF-8 BOM なしが安全。

---

## GitHub Copilot / IntelliCode の補完を無効化する

インストール直後は AI による自動補完が有効になっており、コードを書くたびにグレーのサジェストが表示される。慣れないうちは邪魔に感じることが多いため、必要に応じて無効化する。

### GitHub Copilot を無効化する

拡張機能をインストールしている場合:

**ツール > オプション > GitHub > Copilot**

- 「Enable GitHub Copilot」のチェックを外す

または、エディタ右下のステータスバーにある Copilot アイコンをクリックして一時的に無効化することもできる。

### IntelliCode の自動補完を無効化する

Visual Studio 2022 に標準搭載の AI 補完機能:

**ツール > オプション > IntelliCode > 全般**

- 「C# の自動補完の表示」を「無効」にする

---

## その他のおすすめ設定

| 設定項目 | 場所 | 推奨値 |
|----------|------|--------|
| 自動保存 | ツール > オプション > 環境 > ドキュメント | 「ソース管理に保存するときにドキュメントを保存する」をオン |
| 空白の表示 | 編集 > 詳細 > 空白の表示切り替え | デバッグ中に不要なスペース混入を発見しやすい |
| スクロールバーのマップモード | ツール > オプション > テキスト エディター > すべての言語 > スクロール バー | 動作: マップ モードを使用する（長いファイルの全体把握に便利）|

# 3: ビルドと実行の仕組み

## この記事で学ぶこと

- ビルド・再ビルド・クリーンの違い
- Debug と Release の違いと使い分け
- 出力先フォルダと成果物の確認方法
- F5 と Ctrl+F5 の違い

---

## ビルド・再ビルド・クリーンの違い

**ビルド** メニューから操作できる。

| 操作 | 動作 |
|------|------|
| ビルド（F6） | 変更されたファイルだけをコンパイルする |
| 再ビルド | すべてのファイルを一から再コンパイルする |
| クリーン | ビルド済みの出力ファイル（bin / obj フォルダ）を削除する |

通常は **ビルド** を使えばよい。「なぜか挙動がおかしい」と感じたら **再ビルド** か **クリーン → ビルド** を試す。

---

## Debug と Release の違い

ツールバーのドロップダウン（デフォルトは「Debug」）または **ビルド > 構成マネージャー** で切り替えられる。

| 項目 | Debug | Release |
|------|-------|---------|
| 最適化 | なし（コードが読みやすい） | あり（実行速度が上がる） |
| デバッグ情報（.pdb） | 出力される | 出力されない（設定による） |
| `DEBUG` シンボル | 定義される | 定義されない |
| 実行速度 | 遅め | 速い |
| ファイルサイズ | 大きい | 小さい |

**開発中は Debug、配布するときは Release でビルドする。**

`#if DEBUG` ブロックを使うと、Debug ビルドのみで実行されるコードを書ける。

```csharp
#if DEBUG
Console.WriteLine("デバッグ用ログ");
#endif
```

## 出力先フォルダと成果物

ビルドすると `bin` フォルダに成果物が生成される。

```
プロジェクトフォルダ/
  bin/
    Debug/
      net8.0/
        MyApp.exe       ← 実行ファイル
        MyApp.dll
        MyApp.pdb       ← デバッグ情報
    Release/
      net8.0/
        MyApp.exe
        MyApp.dll
  obj/                  ← ビルド中間ファイル（直接使わない）
```

配布するときは `bin/Release/net8.0/` フォルダの内容を渡す（→ 詳細は記事 10）。

## F5 と Ctrl+F5 の違い

| キー | 動作 |
|------|------|
| F5 | デバッガーをアタッチして実行。ブレークポイントで止まる |
| Ctrl+F5 | デバッガーなしで実行。コンソールアプリは終了後もウィンドウが残る |
| Shift+F5 | デバッグを停止する |

コンソールアプリを F5 で実行すると、すぐウィンドウが閉じてしまうことがある。`Console.ReadLine()` を末尾に書くか、Ctrl+F5 を使うとウィンドウが残る。

---

## よくある落とし穴

- **Debug ビルドで配布する**: 実行は遅く、.pdb ファイルが含まれてしまう。配布前は必ず Release ビルドに切り替える
- **obj フォルダを Git に含める**: `bin/` と `obj/` は `.gitignore` で除外するのが標準。Visual Studio が自動生成する `.gitignore` には含まれている

## NuGet パッケージとは

NuGet は .NET 向けのパッケージ管理システム。外部ライブラリ（Excel 操作・HTTP 通信・ログ出力など）を簡単に追加できる。

業務自動化でよく使うパッケージの例:

| パッケージ名 | 用途 |
|-------------|------|
| ClosedXML | Excel ファイルの読み書き |
| EPPlus | Excel ファイルの読み書き（より高機能） |
| NPOI | Excel / Word ファイルの操作 |
| Newtonsoft.Json | JSON のパース・生成 |
| Selenium.WebDriver | ブラウザ操作の自動化 |
| NLog / Serilog | ログ出力 |

## パッケージのインストール

### GUI から操作する（推奨）

1. ソリューション エクスプローラーでプロジェクトを右クリック
2. **NuGet パッケージの管理** を選択
3. **参照** タブでパッケージ名を検索
4. パッケージを選択して **インストール** ボタンをクリック

### パッケージ マネージャー コンソールから操作する

**ツール > NuGet パッケージ マネージャー > パッケージ マネージャー コンソール**

```powershell
Install-Package ClosedXML
Install-Package ClosedXML -Version 0.102.2   # バージョン指定
```

インストールすると `.csproj` に `<PackageReference>` が追記される:

```xml
<ItemGroup>
  <PackageReference Include="ClosedXML" Version="0.102.2" />
</ItemGroup>
```

## パッケージの更新・削除

### 更新

NuGet パッケージの管理 → **更新プログラム** タブで更新可能なパッケージが一覧表示される。個別に更新するか「すべて更新」を選べる。

### 削除

NuGet パッケージの管理 → **インストール済み** タブで対象パッケージを選択 → **アンインストール** ボタン。

---

## バージョン固定の考え方

業務ツールは長期間同じバージョンで使い続けることが多い。パッケージを更新すると動作が変わる場合があるため、バージョンを固定しておくのが安全。

`.csproj` でバージョンを明示的に指定しておくと、他の環境でも同じバージョンが復元される:

```xml
<PackageReference Include="ClosedXML" Version="0.102.2" />  <!-- 固定 -->
<PackageReference Include="ClosedXML" Version="0.*" />       <!-- マイナーは自動更新 -->
```

---

## パッケージの復元

`.csproj` に記載されたパッケージ情報をもとに、実際のライブラリを `packages` フォルダや NuGet キャッシュから復元する操作。

- ソリューションを開いたとき・ビルド時に自動的に復元される
- 手動で行うには: ソリューションを右クリック → **NuGet パッケージの復元**

## よくある落とし穴

- **インターネット未接続の環境でビルドできない**: NuGet はデフォルトで `nuget.org` からパッケージをダウンロードする。オフライン環境では事前にキャッシュするか、ローカルフィードを設定する必要がある
- **バージョン競合**: パッケージ A が依存するパッケージ C のバージョンと、パッケージ B が依存するバージョンが異なる場合に発生する。エラー一覧に `NU1107` などのコードが出る
- **NuGet キャッシュの破損**: ビルドエラーが解消しないときは **ツール > NuGet パッケージ マネージャー > パッケージ マネージャー設定** の「すべての NuGet キャッシュをクリア」を試す

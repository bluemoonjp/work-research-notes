## ブレークポイントの基本

ブレークポイントとは、プログラムを一時停止させる「目印」。設置した行でデバッガーが処理を止め、変数の状態などを確認できる。

### 設置方法

- コード行の左端の余白をクリック（赤い丸が表示される）
- または行にカーソルを置いて **F9**

F5 で実行すると、ブレークポイントに達した時点で一時停止する。

### 削除・有効/無効の切り替え

- 赤い丸を再クリックで削除
- 右クリック → **ブレークポイントを無効にする** で一時的に無効化

## ステップ実行

一時停止後、1 行ずつ処理を進める操作。

| キー | 動作 |
|------|------|
| F10 | ステップ オーバー。現在の行を実行し、メソッド呼び出しは中に入らない |
| F11 | ステップ イン。メソッド呼び出しの中に入る |
| Shift+F11 | ステップ アウト。現在のメソッドを最後まで実行して呼び出し元に戻る |
| F5 | 次のブレークポイントまで実行を再開する |
| Shift+F5 | デバッグを終了する |

コードの流れを追うときは F10 を基本とし、特定のメソッドの動作を確認したいときだけ F11 で入る。

## 変数を確認するウィンドウ

一時停止中に変数の値を確認できるウィンドウが複数ある。

### ローカル ウィンドウ

現在のスコープにある変数が自動的に一覧表示される。**デバッグ > ウィンドウ > ローカル**。

### ウォッチ ウィンドウ

任意の変数や式を監視できる。変数名を右クリック → **ウォッチに追加** または **デバッグ > ウィンドウ > ウォッチ 1**。式（`list.Count > 0` など）も入力できる。

### イミディエイト ウィンドウ

一時停止中に任意の式を実行・評価できる。**デバッグ > ウィンドウ > イミディエイト**（Ctrl+Alt+I）。

```
? myList.Count      ← 変数の値を表示
? myDate.ToString("yyyy/MM/dd")  ← メソッドを呼び出して結果を確認
```

## 条件付きブレークポイント

特定の条件のときだけ一時停止させる設定。ループの途中など、特定の反復でだけ止めたいときに使う。

ブレークポイントの赤い丸を右クリック → **条件**。

```
i == 50         ← i が 50 のときだけ停止
items.Count > 100
```

---

## ホバーで変数の値を確認する

一時停止中にコード上の変数にマウスを乗せると、その時点での値がポップアップ表示される。配列やオブジェクトは展開して中身を確認できる。

## よくある落とし穴

- **Release ビルドでデバッグしようとする**: コンパイラ最適化によってブレークポイントが期待通りに止まらない。デバッグは必ず Debug ビルドで行う
- **async メソッドのステップ実行**: `await` の行は F11 で入っても直感と異なる動きをする場合がある。非同期の動作確認はログ出力を組み合わせると分かりやすい
- **例外が発生しているのに止まらない**: `try-catch` で例外を握りつぶしているケースが多い。**例外設定**（Ctrl+Alt+E）で「スロー時」にチェックを入れると、catch された例外も捕捉できる

# ファイル IO

## この記事で学ぶこと

- `System.IO` 名前空間の全体像
- `File` クラスの静的メソッドによる簡便な読み書き
- `StreamReader` / `StreamWriter` による逐次処理
- `FileStream` によるバイナリ操作
- `Path` クラスによるパス文字列の操作
- `Directory` クラスによるディレクトリ操作
- `using` を使ったリソース解放パターン
- CSV ファイルの読み書き実例

---

## `System.IO` 名前空間の概要

| クラス | 役割 |
|---|---|
| `File` | テキスト・バイナリファイルの静的ユーティリティ |
| `Directory` | ディレクトリの作成・削除・列挙 |
| `Path` | パス文字列の操作（結合・拡張子取得等） |
| `StreamReader` | テキストファイルの逐次読み取り |
| `StreamWriter` | テキストファイルの逐次書き込み |
| `FileStream` | バイナリファイルのストリーム操作 |
| `BinaryReader` / `BinaryWriter` | プリミティブ型のバイナリ読み書き |

## `File` クラス（静的メソッド）

小〜中サイズのファイルを一括で読み書きする場合に最も手軽です。

```csharp
// 全テキストを一括読み取り
string content = File.ReadAllText("data.txt");

// 行単位で読み取り
string[] lines = File.ReadAllLines("data.txt");

// 全テキストを書き込み（上書き）
File.WriteAllText("output.txt", "Hello, World!");

// 行単位で書き込み
File.WriteAllLines("output.txt", new[] { "Line1", "Line2" });

// 追記
File.AppendAllText("log.txt", "追加行\n");

// ファイルの存在確認
bool exists = File.Exists("data.txt");

// コピー・移動・削除
File.Copy("src.txt", "dst.txt", overwrite: true);
File.Move("old.txt", "new.txt");
File.Delete("temp.txt");
```

**文字コードの指定:** 既定は UTF-8 BOM なしです。Shift-JIS など別の文字コードが必要な場合はオーバーロードで `Encoding` を渡します。

```csharp
using System.Text;
string sjisContent = File.ReadAllText("sjis.txt", Encoding.GetEncoding("shift_jis"));
```

## `StreamReader` / `StreamWriter`（逐次処理）

大きなファイルを一括でメモリに読み込まず、行ごとに処理したい場合に使います。

```csharp
using var reader = new StreamReader("large.txt");
while (!reader.EndOfStream)
{
    string? line = reader.ReadLine();
    // 行ごとに処理
}
```

```csharp
using var writer = new StreamWriter("output.txt", append: false);
writer.WriteLine("1行目");
writer.WriteLine("2行目");
// using ブロックを抜けると自動的に Flush & Close される
```

**`using` が重要な理由:** `StreamReader` / `StreamWriter` は内部でファイルハンドルを保持します。`using` を使わないとファイルが開いたままになり、他のプロセスからアクセスできなくなったり、書き込みがフラッシュされない場合があります。

## `FileStream`（バイナリ）

バイト単位でファイルを操作する場合に使います。画像・音声・バイナリプロトコルなどが対象です。

```csharp
using var fs = new FileStream("data.bin", FileMode.OpenOrCreate, FileAccess.ReadWrite);
byte[] buffer = { 0x01, 0x02, 0x03 };
fs.Write(buffer, 0, buffer.Length);
fs.Seek(0, SeekOrigin.Begin);
int b = fs.ReadByte(); // 0x01
```

テキストファイルを扱う場合は `File` クラスや `StreamReader` / `StreamWriter` の方が適切です。`FileStream` は低レベルな操作が必要なときに限定して使ってください。

## `Path` クラス

パス文字列を OS に依存せず操作するためのユーティリティです。手動で `/` や `\` を連結することは避け、`Path` クラスを使ってください。

```csharp
string dir  = @"C:\Users\user\Documents";
string file = "report.csv";

string full = Path.Combine(dir, file);           // C:\Users\user\Documents\report.csv
string ext  = Path.GetExtension(full);           // .csv
string name = Path.GetFileName(full);            // report.csv
string stem = Path.GetFileNameWithoutExtension(full); // report
string parent = Path.GetDirectoryName(full)!;    // C:\Users\user\Documents

string temp = Path.GetTempFileName();            // 一時ファイルのパス
string rand = Path.GetRandomFileName();          // ランダムなファイル名
```

## `Directory` クラス

```csharp
// ディレクトリ作成（既存でもエラーにならない）
Directory.CreateDirectory(@"C:\temp\output");

// ファイル列挙（サブディレクトリは含まない）
string[] csvFiles = Directory.GetFiles(@"C:\data", "*.csv");

// 再帰的に列挙
string[] allFiles = Directory.GetFiles(@"C:\data", "*.*", SearchOption.AllDirectories);

// サブディレクトリ一覧
string[] dirs = Directory.GetDirectories(@"C:\data");

// 存在確認・削除
bool exists = Directory.Exists(@"C:\temp");
Directory.Delete(@"C:\temp\old", recursive: true);
```

## `using` を使ったリソース解放パターン

### `using` ステートメント（ブロック形式）

```csharp
using (var reader = new StreamReader("file.txt"))
{
    // このブロックを抜けると Dispose() が呼ばれる
    string content = reader.ReadToEnd();
}
```

### `using` 宣言（C# 8+、スコープ形式）

```csharp
using var reader = new StreamReader("file.txt");
// メソッド（またはブロック）を抜けると Dispose() が呼ばれる
string content = reader.ReadToEnd();
```

`using` 宣言はネストが浅くなり読みやすいですが、スコープがメソッド全体になる点に注意してください。明示的に早期解放したい場合はブロック形式を使います。

## CSV 読み書きの実例

### CSV 読み取り

```csharp
var records = new List<string[]>();
using var reader = new StreamReader("input.csv");
while (!reader.EndOfStream)
{
    string? line = reader.ReadLine();
    if (line is null) continue;
    records.Add(line.Split(','));
}
```

### CSV 書き込み

```csharp
var rows = new List<string[]>
{
    new[] { "名前", "点数" },
    new[] { "Alice", "90" },
    new[] { "Bob",   "75" },
};
using var writer = new StreamWriter("output.csv", append: false, encoding: new System.Text.UTF8Encoding(encoderShouldEmitUTF8Identifier: true));
foreach (var row in rows)
{
    writer.WriteLine(string.Join(",", row));
}
```

**注意:** カンマや改行を含むフィールドを正しく扱うには、各フィールドをダブルクォートで囲む処理が必要です。本格的な CSV 処理には CsvHelper などのライブラリの利用を検討してください。

## よくある落とし穴

- **`using` を忘れる**: `StreamReader` / `StreamWriter` / `FileStream` は必ず `using` で囲むか、`try/finally` で `Dispose()` を確実に呼んでください。
- **相対パスの基準点**: 相対パスはアプリケーションの作業ディレクトリ（`Directory.GetCurrentDirectory()`）を基準にします。`AppDomain.CurrentDomain.BaseDirectory` と異なる場合があります。
- **文字コードの不一致**: Windows 環境の古いファイルは Shift-JIS の場合があります。読み取りに失敗したり文字化けする場合は文字コードを確認してください。
- **同時アクセス**: 同一ファイルを複数の `StreamWriter` から同時に書き込むと例外が発生します。ロック機構（`lock` や `Mutex`）を使うか、1 つのインスタンスを共有してください。

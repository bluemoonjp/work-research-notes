# 1: 型と変数宣言

## この記事で学ぶこと

- C# の組み込み型（`int`, `double`, `decimal`, `bool`, `char`, `string`, `object` など）
- `var` による型推論（コンパイル時推論であることの意味）
- 値型と参照型の違い（スタック／ヒープ）
- `const` と `readonly` の使い分け
- `string` の不変性と `StringBuilder` の使いどころ
- タプル `(int x, int y)` の使い方

## 組み込み型

C# の組み込み型は、.NET の型のエイリアス（別名）です。

| C# キーワード | .NET 型 | サイズ / 範囲 | 用途 |
|-------------|---------|------------|------|
| `bool` | `System.Boolean` | `true` / `false` | 真偽値 |
| `byte` | `System.Byte` | 0〜255（符号なし 8 bit） | バイナリデータ |
| `int` | `System.Int32` | −2,147,483,648〜2,147,483,647 | 一般的な整数 |
| `long` | `System.Int64` | 約 ±9.2 × 10¹⁸ | 大きな整数 |
| `float` | `System.Single` | 7 桁精度（32 bit 浮動小数点） | 精度より速度を優先する場合 |
| `double` | `System.Double` | 15〜17 桁精度（64 bit 浮動小数点） | 一般的な浮動小数点 |
| `decimal` | `System.Decimal` | 28〜29 桁精度（128 bit）| **金額・税率など誤差が許されない計算** |
| `char` | `System.Char` | Unicode 文字 1 文字 | 文字単体（シングルクォート `'A'`） |
| `string` | `System.String` | 文字列（参照型）| テキスト |
| `object` | `System.Object` | すべての型の基底 | 型が不明な値（多用は避ける） |

### `decimal` を使う場面

```csharp
double price = 0.1 + 0.2;    // 0.30000000000000004（浮動小数点誤差）
decimal price2 = 0.1m + 0.2m; // 0.3（正確）

// m サフィックスで decimal リテラルを表す
decimal tax = 0.10m;
```

金融・会計系の計算では必ず `decimal` を使ってください。

## `var` による型推論

`var` はコンパイル時に型を推論するキーワードです。**実行時に型が変わる動的型付けではありません。**

```csharp
var count = 42;          // int として推論される
var name = "Alice";      // string として推論される
var items = new List<int>(); // List<int> として推論される

// count = "hello";  // コンパイルエラー: var で推論された型は固定
```

### `var` の使い分けの基準

| 使う場面 | 理由 |
|---------|------|
| `new` 式（`var x = new SomeType()`） | 型名の重複を避けられる |
| LINQ の結果 | 返り値の型が複雑になりやすい |
| 型が右辺から明らかな場合 | 読みやすさが向上する |
| 型が右辺から不明な場合 | **明示的な型宣言を使う**（可読性のため） |

```csharp
// 推奨: 型が右辺から明らか
var user = new User("Alice");
var list = new List<string>();

// 非推奨: 型が右辺から不明（ReadLine は string? を返すが一見わかりにくい）
string? input = Console.ReadLine();  // var より明示的な型宣言のほうが明確
```

## 値型と参照型

### 値型（Value Type）

- **スタック**に格納される（小さく、高速）
- 代入すると値がコピーされる
- `struct`、`enum`、プリミティブ型（`int`, `bool`, `char` など）

```csharp
int a = 10;
int b = a;  // 値がコピーされる
b = 20;
Console.WriteLine(a); // → 10（a は影響を受けない）
```

### 参照型（Reference Type）

- **ヒープ**にオブジェクトが格納され、変数はその参照（アドレス）を持つ
- 代入すると参照がコピーされる（同じオブジェクトを指す）
- `class`、`string`、配列、インターフェース

```csharp
var list1 = new List<int> { 1, 2, 3 };
var list2 = list1;  // 参照がコピーされる（同じリストを指す）
list2.Add(4);
Console.WriteLine(list1.Count); // → 4（list1 も影響を受ける）
```

### よくある落とし穴: `string` は参照型だが不変

`string` は参照型ですが、**変更操作は常に新しいオブジェクトを返します**（不変・イミュータブル）。

```csharp
string s1 = "Hello";
string s2 = s1;
s1 = s1 + " World";  // 新しい string オブジェクトが生成される
Console.WriteLine(s2); // → "Hello"（s2 は影響を受けない）
```

## `const` と `readonly`

| | `const` | `readonly` |
|--|---------|-----------|
| 評価タイミング | **コンパイル時** | 実行時（コンストラクタで設定可能） |
| 使える型 | プリミティブ型・`string` のみ | すべての型 |
| インスタンスごとに違う値 | 不可 | 可（インスタンスフィールドとして） |
| `static` | 暗黙的に `static` | 明示的に `static` を付けられる |

```csharp
public class Config
{
    public const int MaxRetry = 3;               // コンパイル時定数
    public const string AppName = "MyApp";

    public readonly DateTime CreatedAt;          // インスタンスごとの読み取り専用
    public static readonly string Version = LoadVersion(); // 実行時に一度だけ初期化

    public Config()
    {
        CreatedAt = DateTime.Now; // コンストラクタで設定できる
    }

    static string LoadVersion() => "1.0.0";
}
```

### 使い分けの基準

- **`const`**: 絶対に変わらない定数（数学定数、アプリ固定値など）
- **`readonly`**: 初期化後は変更されないが、実行時の値に依存する場合

## `string` と `StringBuilder`

### `string` の不変性と連結コスト

```csharp
// NG: ループ内で + 連結すると O(n²) のコピーが発生する
string result = "";
for (int i = 0; i < 1000; i++)
{
    result += i.ToString(); // 毎回新しい string が生成される
}
```

### `StringBuilder` を使う

```csharp
using System.Text;

// OK: StringBuilder はバッファを内部で再利用する
var sb = new StringBuilder();
for (int i = 0; i < 1000; i++)
{
    sb.Append(i);
}
string result = sb.ToString();
```

### 使い分けの基準

| 使う場面 | 推奨 |
|---------|------|
| 文字列が 4〜5 回以下の連結 | `string` + `$""` 補間 |
| ループ内での連結、大量の結合 | `StringBuilder` |
| 最終的に一度だけ文字列が必要 | `string.Join()` |

## タプル

複数の値をまとめて返したい場合に使います。専用クラスを作るほどでもない一時的な組み合わせに適しています。

```csharp
// 名前付きタプル
(int Min, int Max) range = (1, 100);
Console.WriteLine(range.Min); // → 1

// メソッドの戻り値としてのタプル
(string Name, int Age) GetUser() => ("Alice", 30);

var user = GetUser();
Console.WriteLine($"{user.Name}, {user.Age}歳");

// 分解（デコンストラクション）
var (name, age) = GetUser();
Console.WriteLine($"{name}, {age}歳");
```

### よくある落とし穴

- タプルは値型（`System.ValueTuple`）です。参照型のクラスと混同しないように注意してください。
- 複数の場所で同じ「組み合わせ」を使う場合は、専用のクラスや `record` 型を定義した方が意図が伝わりやすくなります。

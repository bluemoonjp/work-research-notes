## 配列（`T[]`）

配列は**固定長**で、インデックスによるアクセスが O(1) の最も基本的なコレクションです。

```csharp
int[] numbers = new int[5];       // 長さ 5、全要素 0
int[] primes  = { 2, 3, 5, 7 };  // 初期化子構文
```

**特性:**
- 宣言後に長さを変更できない
- スタックではなくヒープに確保される（値型の要素も含む）
- 多次元配列（`int[,]`）とジャグ配列（`int[][]`）がある

**他言語との違い:** Python のリストや JavaScript の配列は動的サイズですが、C# の `T[]` は固定長です。動的サイズが必要なら `List<T>` を使います。

## `List<T>`（動的配列）

内部的に配列を持ち、容量超過時に自動的に拡張します。最もよく使う汎用コレクションです。

```csharp
var list = new List<string>();
list.Add("Alice");
list.AddRange(new[] { "Bob", "Carol" });
list.Remove("Bob");
list.Insert(0, "先頭");
int count = list.Count; // 要素数（Length ではなく Count）
```

**よくある落とし穴:** `Array.Length` は `Length` プロパティですが、`List<T>` は `Count` です。混同しないよう注意してください。

## `Dictionary<TKey, TValue>`

キーでバリューを検索する連想コレクションです。キーの検索は平均 O(1)（ハッシュテーブル実装）。

```csharp
var scores = new Dictionary<string, int>
{
    ["Alice"] = 90,
    ["Bob"]   = 75,
};

scores["Carol"] = 85;          // 追加
scores["Alice"] = 95;          // 上書き

// キーが存在するか確認してから取得
if (scores.TryGetValue("Dave", out int score))
{
    Console.WriteLine(score);
}
// キーなしで直接アクセスすると KeyNotFoundException
```

**使い分けの基準:** キーで引きたいなら `Dictionary`、順番や位置で引きたいなら `List` または配列を選択します。

## `HashSet<T>`

重複を持たないコレクションです。`Contains` が O(1) のため、「存在チェック」に向いています。

```csharp
var set = new HashSet<int> { 1, 2, 3, 2, 1 };
// set は { 1, 2, 3 } — 重複は自動除去

set.Add(4);        // 追加
set.Remove(2);     // 削除
bool has = set.Contains(3); // 高速検索

// 集合演算
var other = new HashSet<int> { 3, 4, 5 };
set.IntersectWith(other);  // 積集合
set.UnionWith(other);      // 和集合
set.ExceptWith(other);     // 差集合
```

## `Queue<T>` と `Stack<T>`

### Queue（先入れ先出し: FIFO）

```csharp
var queue = new Queue<string>();
queue.Enqueue("First");
queue.Enqueue("Second");
string item = queue.Dequeue(); // "First" を取り出す
string peek = queue.Peek();    // 取り出さずに先頭を確認
```

### Stack（後入れ先出し: LIFO）

```csharp
var stack = new Stack<int>();
stack.Push(1);
stack.Push(2);
int top = stack.Pop();   // 2 を取り出す
int peek = stack.Peek(); // 取り出さずに確認
```

## インターフェース階層

コレクションを受け取るメソッドの引数型には、必要最小限のインターフェースを使うのが設計上の慣習です。

```
IEnumerable<T>          foreach で列挙できる（最小）
  └─ ICollection<T>     Count, Add, Remove, Contains を持つ
       └─ IList<T>      インデックスアクセス（[i]）を持つ
            └─ T[]
            └─ List<T>
```

```csharp
// 読み取りだけなら IEnumerable<T> で受け取る
void Print(IEnumerable<string> items)
{
    foreach (var item in items) Console.WriteLine(item);
}
```

## コレクション式（C# 12+）

角括弧 `[...]` でコレクションを生成できます。型推論により `List<T>`、配列、`Span<T>` 等に対応します。

```csharp
int[] arr   = [1, 2, 3];
List<int> l = [1, 2, 3];

// スプレッド演算子でコレクションを展開
int[] a = [1, 2];
int[] b = [3, 4];
int[] c = [..a, ..b]; // [1, 2, 3, 4]
```

## `Span<T>` と `Memory<T>`（パフォーマンス目的）

`Span<T>` は配列や文字列の連続したメモリ領域を**コピーなし**で参照するための構造体です。ヒープ割り当てを避けてパフォーマンスを上げたいときに使います。

- `Span<T>`: スタック上でのみ使用可能（async メソッドやフィールドには使えない）
- `Memory<T>`: ヒープに置けるため async でも使用可能

```csharp
int[] data = { 1, 2, 3, 4, 5 };
Span<int> slice = data.AsSpan(1, 3); // インデックス 1 から 3 要素（コピーなし）
```

通常の業務コードでは `List<T>` や配列で十分です。`Span<T>` は大量データの処理や文字列パース等、パフォーマンスクリティカルな場面で検討してください。

---

## よくある落とし穴

- **`foreach` 中に `List` を変更しない**: `InvalidOperationException` が発生します。変更が必要なら `for` ループかコピーを使ってください。
- **`Dictionary` のキーは `null` 不可**: キーに null を渡すと `ArgumentNullException` が発生します。
- **`HashSet` の順序は保証されない**: 順序が必要なら `SortedSet<T>` または `List<T>` を使ってください。

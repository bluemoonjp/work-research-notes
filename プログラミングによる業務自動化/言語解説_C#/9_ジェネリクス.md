# ジェネリクス (Generics)

## この記事で学ぶこと

- ジェネリクスの目的（型安全性とコード再利用）
- ジェネリッククラス・ジェネリックメソッドの定義方法
- 型制約（`where` 句）の種類と使い方
- 共変性・反変性（`out` / `in`）の概念
- よく使うジェネリック型（`List<T>` / `Dictionary<TKey,TValue>` など）

## ジェネリクスの目的

ジェネリクスを使うと、**型をパラメータとして受け取るクラスやメソッド**を定義できます。

```csharp
// NG: object 型を使うと型安全性がない（キャスト失敗のリスク）
object box = "文字列";
int n = (int)box; // 実行時エラー: InvalidCastException

// OK: ジェネリクスで型を固定
Box<string> strBox = new Box<string>("文字列");
// strBox.Value は string として扱われる（キャスト不要）
```

**メリット:**
- コンパイル時に型チェックされるため、実行時の型エラーを防げる
- キャストが不要になりコードが簡潔になる
- `int`・`double` などの値型を `object` にボックス化するコストが不要

## ジェネリッククラスの定義

型パラメータは慣習的に `T`（Type の略）や `TKey` / `TValue` のように命名します。

```csharp
public class Box<T>
{
    public T Value { get; set; }

    public Box(T value)
    {
        Value = value;
    }
}

// 使用例
var intBox    = new Box<int>(42);
var stringBox = new Box<string>("hello");
```

複数の型パラメータも定義できます。

```csharp
public class Pair<TFirst, TSecond>
{
    public TFirst  First  { get; }
    public TSecond Second { get; }

    public Pair(TFirst first, TSecond second)
    {
        First  = first;
        Second = second;
    }
}
```

## ジェネリックメソッドの定義

メソッド単位で型パラメータを定義することもできます。

```csharp
public static T Max<T>(T a, T b) where T : IComparable<T>
{
    return a.CompareTo(b) >= 0 ? a : b;
}

// 呼び出し: 型推論により型引数を省略できる
int    m1 = Max(3, 7);          // T = int と推論される
string m2 = Max("abc", "xyz"); // T = string と推論される
```

## 型制約（where 句）

型パラメータが満たすべき条件を `where` で指定できます。  
制約を付けることで、その型のメソッドやプロパティをジェネリック実装の中で安全に呼び出せます。

| 制約 | 意味 |
|---|---|
| `where T : class` | T は参照型（null 許容） |
| `where T : struct` | T は値型（null 不可） |
| `where T : notnull` | T は null 非許容（C# 8 以降） |
| `where T : new()` | T はパラメータなしコンストラクタを持つ |
| `where T : SomeClass` | T は `SomeClass` またはそのサブクラス |
| `where T : IComparable<T>` | T は `IComparable<T>` を実装する |
| `where T : class, new()` | 複数制約の組み合わせ |

```csharp
// new() 制約: T のインスタンスを内部で生成できる
public static T CreateInstance<T>() where T : new()
{
    return new T();
}

// IComparable<T> 制約: 比較メソッドを呼べる
public static T Clamp<T>(T value, T min, T max) where T : IComparable<T>
{
    if (value.CompareTo(min) < 0) return min;
    if (value.CompareTo(max) > 0) return max;
    return value;
}
```

### よくある落とし穴: 制約なしでは T のメンバーを呼べない

```csharp
// NG: T に CompareTo があるかどうか不明なのでコンパイルエラー
public static T Max<T>(T a, T b)
{
    return a.CompareTo(b) >= 0 ? a : b; // エラー
}

// OK: IComparable<T> 制約を追加する
public static T Max<T>(T a, T b) where T : IComparable<T>
{
    return a.CompareTo(b) >= 0 ? a : b;
}
```

## 共変性・反変性（out / in）

インターフェイスやデリゲートのジェネリック型パラメータに `out` / `in` を付けると、型の代入互換性が拡張されます。

### 共変性（out）: より派生した型を返せる

```csharp
// IEnumerable<T> は out T（共変）
IEnumerable<string> strings = new List<string> { "a", "b" };
IEnumerable<object> objects = strings; // string は object のサブクラスなので代入可能
```

`out` を付けると「出力専用（返すだけ）」になります。メソッドの戻り値や読み取り専用プロパティにのみ使えます。

### 反変性（in）: より基底な型を受け取れる

```csharp
// Action<T> は in T（反変）
Action<object> printObj = obj => Console.WriteLine(obj);
Action<string> printStr = printObj; // object を受け取れるなら string も受け取れる
```

`in` を付けると「入力専用（受け取るだけ）」になります。メソッドのパラメータにのみ使えます。

> 実務では `IEnumerable<out T>` / `IReadOnlyList<out T>` などの既存インターフェイスで自然に使われています。自分でインターフェイスを定義するときに意識する場面は少ないですが、「なぜ `IEnumerable<string>` を `IEnumerable<object>` に代入できるのか」の理由として知っておくと役立ちます。

## よく使うジェネリック型

| 型 | 用途 |
|---|---|
| `List<T>` | 可変長リスト。`ArrayList` の型安全版 |
| `Dictionary<TKey, TValue>` | キーと値のマップ |
| `HashSet<T>` | 重複なし集合 |
| `Queue<T>` | FIFO キュー |
| `Stack<T>` | LIFO スタック |
| `IEnumerable<T>` | 読み取り専用シーケンス（遅延評価の基盤） |
| `IReadOnlyList<T>` | インデックスアクセス可能な読み取り専用コレクション |
| `Nullable<T>` / `T?` | 値型に null を許容する |
| `Task<T>` | 非同期処理の結果を持つ型 |
| `Func<T, TResult>` | 引数を受けて値を返すデリゲート |
| `Action<T>` | 引数を受けて void を返すデリゲート |

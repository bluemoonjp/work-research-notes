## インターフェースの定義と実装

`interface` はメンバーのシグネチャ（型・名前・引数）だけを定義する契約です。実装はそのインターフェースを実装するクラス・構造体が提供します。

```csharp
interface IShape
{
    double Area { get; }
    double Perimeter { get; }
    string Describe();
}

class Circle : IShape
{
    public double Radius { get; }

    public Circle(double radius) => Radius = radius;

    // インターフェースのメンバーをすべて実装する必要がある
    public double Area      => Math.PI * Radius * Radius;
    public double Perimeter => 2 * Math.PI * Radius;
    public string Describe() => $"Circle(r={Radius})";
}
```

インターフェースを実装するクラスは、すべてのメンバーを実装しなければなりません（デフォルト実装がある場合を除く）。

## デフォルトインターフェースメソッド（C# 8+）

C# 8 からインターフェースにデフォルト実装を持つメソッドを定義できます。既存のインターフェースに影響を与えず新しいメソッドを追加するときに有用です。

```csharp
interface IShape
{
    double Area { get; }
    double Perimeter { get; }

    // デフォルト実装（実装クラスで省略可）
    string Describe() => $"Area={Area:F2}, Perimeter={Perimeter:F2}";
}
```

> **使い分けの基準:** デフォルト実装はあくまで後方互換のための機能です。新規設計では実装クラスに明示的に実装するほうがコードの意図が明確になります。

## 複数インターフェースの実装

C# は多重継承（クラスの複数継承）をサポートしませんが、**複数のインターフェースは実装できます**。

```csharp
interface IPrintable
{
    void Print();
}

interface ISerializable
{
    string Serialize();
}

// 複数インターフェースをカンマ区切りで実装
class Document : IPrintable, ISerializable
{
    public string Content { get; }

    public Document(string content) => Content = content;

    public void Print()         => Console.WriteLine(Content);
    public string Serialize()   => $"{{\"content\":\"{Content}\"}}";
}
```

## 抽象クラスとインターフェースの使い分け

| 特性 | interface | abstract class |
|---|---|---|
| 多重実装 | 可（複数 OK） | 不可（単一継承） |
| 状態（フィールド） | 不可 | 可 |
| コンストラクタ | 不可 | 可 |
| デフォルト実装 | 可（C# 8+） | 可 |
| アクセス修飾子 | public のみ（デフォルト） | 自由 |

**使い分けの基準:**

- **インターフェース**: 「できること（能力）」を表す契約に使います。`IDisposable`（解放できる）・`IEnumerable`（列挙できる）など、異なるクラス階層に横断的に適用できます。
- **抽象クラス**: 「is-a 関係」で共通の状態・実装を持つ基底型に使います。例: `Animal`（犬・猫の共通基底）。テンプレートメソッドパターンにも適しています。

## 継承と virtual / override

`virtual` メソッドはサブクラスで `override` できます。`override` しないとサブクラスは親の実装を使います。

```csharp
class Animal
{
    public string Name { get; }

    public Animal(string name) => Name = name;

    // virtual: サブクラスでオーバーライド可
    public virtual string Speak() => "...";

    public override string ToString() => $"{GetType().Name}({Name})";
}

class Dog : Animal
{
    public Dog(string name) : base(name) { }

    // override: 親の実装を上書き
    public override string Speak() => "ワン！";
}

class GuideDog : Dog
{
    public GuideDog(string name) : base(name) { }

    // sealed override: これ以上オーバーライドさせない
    public sealed override string Speak() => "静かに働きます";
}
```

> **よくある落とし穴:** `virtual` / `override` を付け忘れると、基底クラスの参照変数からサブクラスのメソッドが呼ばれません（`new` による隠蔽になります）。ポリモーフィズムを意図する場合は必ず `virtual` + `override` を使いましょう。

## base キーワード

`base` は親クラスのメンバーにアクセスするためのキーワードです。

```csharp
class ElectricDog : Dog
{
    public ElectricDog(string name) : base(name) { } // 親のコンストラクタを呼ぶ

    public override string Speak()
    {
        var parentSpeech = base.Speak(); // 親の Speak() を呼ぶ
        return $"{parentSpeech}（電子音）";
    }
}
```

## 多態性（ポリモーフィズム）

基底クラス型（またはインターフェース型）の変数にサブクラスのインスタンスを代入し、統一的に扱えます。呼び出されるメソッドは実行時の実際の型に基づいて決まります（動的ディスパッチ）。

```csharp
Animal[] animals = { new Dog("ポチ"), new Cat("タマ"), new GuideDog("ガイド") };

foreach (var animal in animals)
    Console.WriteLine($"{animal}: {animal.Speak()}"); // 実際の型のメソッドが呼ばれる
```

## is / as 演算子

`is` は型チェック、`as` はキャスト（失敗時は `null` を返す）に使います。

```csharp
object obj = new Dog("ポチ");

// is: 型チェック（パターンマッチング付きが推奨）
if (obj is Dog dog)
{
    Console.WriteLine(dog.Speak()); // 型安全にアクセスできる
}

// as: キャスト（失敗時は null）
var cat = obj as Cat;
if (cat is null)
    Console.WriteLine("Dog を Cat にキャストできませんでした");
```

> **よくある落とし穴:** `(Dog)obj` のような直接キャストは、型が合わない場合に `InvalidCastException` を投げます。安全にダウンキャストするには `is` パターンマッチングか `as` + null チェックを使いましょう。

# 言語別解説: Excel操作を比較する

## 概要比較

| 観点 | VBA | C# |
|---|---|---|
| 実行環境 | Excel 内（VBE にモジュールを貼り付け） | 外部プログラム（ClosedXML で Excel を操作） |
| データアクセス | `ThisWorkbook.Worksheets(...)` で直接参照 | `XLWorkbook(filePath)` でファイルを開く |
| フィルタ・集計 | For ループ＋ If 文 | LINQ（`Where` / `Join` / `GroupBy`） |
| データ型 | Variant で柔軟に扱える | `record` / `class` で型安全 |

---

## プログラム1: 単純なWhere条件

### VBA

```vba
If Year(CDate(発売日)) = Year(Now) And Right(型番, 2) = "HG" Then
```

`Right(文字列, 文字数)` で末尾の文字を取得する。
日付列は `IsDate` でガードしてから `CDate` でキャストするのが安全。

### C#

```csharp
.Where(m => m.発売日.Year == DateTime.Now.Year && m.型番.EndsWith("HG"))
```

`EndsWith` は組み込みメソッド。LINQ の `Where` で条件を関数として渡すため、
追加の変数宣言なしに 1 行で記述できる。

---

## プログラム2: ジョイントグループ化

### VBA の Join アプローチ

1. 商品マスタを `Scripting.Dictionary` に読み込む（key = 商品コード）
2. 注文を 1 行ずつループし、Dictionary からマスタ情報を引く

Dictionary の存在確認は `dic.Exists(key)` で行う。
VBA には LINQ の `Join` に相当する組み込み機能がないため、Dictionary による手動ルックアップが標準的な実装パターン。

### C# の Join アプローチ

```csharp
注文データ.Join(
    マスタデータ,
    o => o.商品コード,    // 注文側のキー
    m => m.商品コード,    // マスタ側のキー
    (o, m) => new { ... } // 結合後の形
)
```

SQL の INNER JOIN と同じ動作。マスタに存在しない商品コードの注文行は自動的に除外される。

### グループ化と集計

**VBA**: 第 2 の Dictionary で `"yyyy/mm_商品コード"` を複合キーとして集計を累積する。

**C#**: `GroupBy(r => new { r.販売年月, r.商品コード })` でグループを作り、
`.Select(g => new { 売上個数 = g.Sum(...), ... })` で集計値を計算する。

### ソート

**VBA**: バブルソートを手動実装する。`販売年月` の文字列比較（`>` / `<`）と利益額の数値比較を組み合わせる。
`yyyy/mm` 形式は文字列として辞書順ソートしても時系列順になる。

**C#**: `OrderBy(r => r.販売年月).ThenByDescending(r => r.利益額)` の 1 行で表現できる。

## プログラム3: 複雑なWhere条件

### VBA

```vba
If (仕入元 = "A社" And 調達区分 = "国内調達") Or _
   (仕入元 = "B社" And 調達区分 = "受注生産") Then
```

`Or` / `And` の優先順位は `And` が高いため、上記の意図どおりに評価される。
行が長くなる場合は `_`（行継続文字）で折り返す。

### C#

```csharp
.Where(m =>
    (m.仕入元 == "A社" && m.調達区分 == "国内調達") ||
    (m.仕入元 == "B社" && m.調達区分 == "受注生産"))
```

`&&` が `||` より優先されるため、括弧の有無によらず意図どおりに評価される。
括弧を書いておくと OR の各節が明確になる。

## プログラム4: 1〜3 まとめ

### VBA

```vba
Sub 実行_1から3まとめ()
    Call 実行_単純なWhere条件
    Call 実行_ジョイントグループ化
    Call 実行_複雑なWhere条件
End Sub
```

各モジュールは独立しており、VBE に 4 モジュールすべてをインポートすれば `Call` で呼び出せる。

### C#

各プログラムのデータ定義クラスと処理クラスを 1 ファイルに統合し、
`Main()` から `Prog1Processor.Run`・`Prog2Processor.Run`・`Prog3Processor.Run` を順に呼ぶ。

ClosedXML ではブックを 1 回だけ開いて最後に 1 回 `Save()` するため、
ファイルの読み書きが 3 回から 1 回に削減される。

## 変数スコープの違い

**VBA**: `Dim` は関数スコープ（Sub / Function 全体）。ループ内に `Dim` を書いても有効範囲は変わらないため、慣習として Sub の先頭にまとめる。

**C#**: `var` や型名による宣言はブロックスコープ。ループ内で宣言した変数はループ外からは見えない。

---

## Excel セル書式の設定

**VBA**: セル個別に `.NumberFormat = "0.00%"` を設定する。

**C#（ClosedXML）**: `sheet.Cell(row, col).Style.NumberFormat.Format = "0.00%"` で同等の書式を設定できる。

# 業務システムからExcelダウンロードして集計

## コードバリアント

| ファイル | ログイン情報の管理方法 | 用途 |
|---|---|---|
| `1_業務システムからExcelダウンロードして集計_CSHARP.cs` | コード内の定数 | ローカル検証・学習 |
| `2_業務システムからExcelダウンロードして集計_DPAPI_CSHARP.cs` | Windows DPAPI でこの PC に暗号化保存 | 本番運用（配布用） |

## 処理の目的

ログインが必要な社内 Web システムにブラウザでアクセスし、売上データ（Excel）をダウンロードする。
ダウンロードしたデータを別の Excel ファイルに転記し、カテゴリ別の集計シートを作成する。

## 想定シナリオ

- 社内 Web システムの売上レポート画面にログインする
- 「エクスポート」ボタンをクリックして Excel（.xlsx）をダウンロードする
- ダウンロードした Excel の中身を読み込み、カテゴリ別に集計して新しい Excel に出力する

コード内の URL・セレクタ・ログイン情報はすべてダミー。実環境では定数を差し替える。

## 前提条件

### NuGet パッケージ

| パッケージ | 用途 | ライセンス | バリアント |
|---|---|---|---|
| `Microsoft.Playwright` | ブラウザ操作 | Apache 2.0（商用無料） | 1・2 共通 |
| `ClosedXML` | Excel 読み書き | MIT（商用無料） | 1・2 共通 |
| `System.Security.Cryptography.ProtectedData` | Windows DPAPI 暗号化 | MIT（商用無料） | 2 のみ |

### 初回セットアップ

NuGet パッケージをインストール後、Chromium ブラウザのバイナリを取得する（初回のみ）:

```
dotnet tool install --global Microsoft.Playwright.CLI
playwright install chromium
```

## 入力 / 出力

| 区分 | バリアント 1 | バリアント 2 |
|---|---|---|
| ログイン情報 | コード内の定数 | 初回起動時に入力 → DPAPI 暗号化保存 |
| ダウンロード | 業務システムが出力する売上 Excel（.xlsx） | 同左 |
| 出力 | `集計結果.xlsx`（生データシート＋カテゴリ集計シート） | 同左 |

### バリアント 2 の利用手順

1. 初回起動時: コンソールにログイン ID とパスワードを入力する
2. 入力した認証情報は `%APPDATA%\SalesAutoTool\credentials.dat` に DPAPI で暗号化保存される
3. 2回目以降は入力不要。保存ファイルから自動的に復号して使用される
4. 認証情報を変更したい場合は `credentials.dat` を削除して再起動する

## ダウンロード Excel の想定フォーマット

| 列 | 内容 | 型 |
|---|---|---|
| A | 注文日 | 文字列（例: 2024-06-01） |
| B | 商品名 | 文字列 |
| C | カテゴリ | 文字列 |
| D | 単価 | 数値 |
| E | 数量 | 整数 |

1行目はヘッダ行、2行目以降がデータ行。

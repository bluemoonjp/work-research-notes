# 2: 制御フロー

## この記事で学ぶこと

- `if` / `else if` / `else` と三項演算子
- `for` / `foreach` / `while` / `do-while` の使い分け
- C# 8+ の switch 式とパターンマッチング（型パターン、when 句、位置パターン、プロパティパターン）
- `break` / `continue` / `return` の使い方
- ガード節（早期リターン）パターンの推奨

## if / else if / else

基本的な構文は多くの言語と共通です。

```csharp
int score = 75;

if (score >= 90)
{
    Console.WriteLine("A");
}
else if (score >= 70)
{
    Console.WriteLine("B");
}
else
{
    Console.WriteLine("C");
}
```

### 三項演算子

1 行で書ける場合に有効ですが、ネストは可読性を損ないます。

```csharp
string grade = score >= 70 ? "合格" : "不合格";
```

## ループ

### for

インデックスが必要な場合に使います。

```csharp
for (int i = 0; i < 5; i++)
{
    Console.WriteLine(i);
}
```

### foreach

コレクションを先頭から順番に処理する場合に使います。インデックスが不要なら `for` よりも `foreach` を優先してください。

```csharp
string[] fruits = { "apple", "banana", "cherry" };
foreach (string fruit in fruits)
{
    Console.WriteLine(fruit);
}
```

インデックスも必要な場合は `LINQ` の `Select` と組み合わせるか、`for` を使います。

```csharp
// インデックス付き foreach（LINQ を使う場合）
foreach (var (item, index) in fruits.Select((v, i) => (v, i)))
{
    Console.WriteLine($"{index}: {item}");
}
```

### while / do-while

```csharp
// while: 条件が最初から false の場合はループに入らない
int n = 0;
while (n < 3)
{
    Console.WriteLine(n);
    n++;
}

// do-while: 必ず 1 回は実行される
do
{
    Console.WriteLine("少なくとも 1 回実行");
} while (false);
```

## switch 式（C# 8+）とパターンマッチング

C# 8 で導入された switch 式は、従来の switch 文より簡潔に書けます。

### 従来の switch 文（C# 7 以前）

```csharp
string GetDayType(DayOfWeek day)
{
    switch (day)
    {
        case DayOfWeek.Saturday:
        case DayOfWeek.Sunday:
            return "休日";
        default:
            return "平日";
    }
}
```

### switch 式（C# 8+）

```csharp
string GetDayType(DayOfWeek day) => day switch
{
    DayOfWeek.Saturday or DayOfWeek.Sunday => "休日",
    _ => "平日"  // _ はデフォルト（ワイルドカード）
};
```

switch 式はすべてのケースを網羅しなければコンパイル警告が出ます。これにより「考慮漏れ」をコンパイル時に検出できます。

### パターンマッチング

#### 型パターン

```csharp
object obj = 42;

if (obj is int n)  // obj が int 型なら変数 n に格納
{
    Console.WriteLine($"整数: {n * 2}");
}
```

#### when 句（条件付きパターン）

```csharp
string Classify(int n) => n switch
{
    < 0 => "負数",
    0 => "ゼロ",
    > 0 and <= 100 => "1〜100",
    _ => "100超"
};
```

#### プロパティパターン

```csharp
record Point(int X, int Y);

string DescribePoint(Point p) => p switch
{
    { X: 0, Y: 0 } => "原点",
    { X: 0 }       => "Y 軸上",
    { Y: 0 }       => "X 軸上",
    { X: > 0, Y: > 0 } => "第 1 象限",
    _              => "その他"
};
```

#### 位置パターン（record や分解可能な型）

```csharp
string DescribePoint2(Point p) => p switch
{
    (0, 0) => "原点",
    (0, _) => "Y 軸上",
    (_, 0) => "X 軸上",
    _      => "その他"
};
```

### よくある落とし穴

- switch 式はすべての入力に対して結果を返す必要があります。`_` のデフォルトケースを忘れると、実行時に `SwitchExpressionException` がスローされます。
- 従来の switch 文と switch 式を混同しないように注意してください。switch 式はアロー `=>` と末尾のカンマを使います。

## break / continue / return

```csharp
for (int i = 0; i < 10; i++)
{
    if (i == 3) continue;  // この反復をスキップ
    if (i == 7) break;     // ループを終了
    Console.WriteLine(i);  // 0, 1, 2, 4, 5, 6 が出力される
}
```

`return` はメソッドを即座に終了します。後述のガード節パターンで活用します。

## ガード節（早期リターン）パターン

ネストを深くせず、異常ケース（前提条件違反など）を先にリターンする書き方です。

```csharp
// NG: 深いネスト（「矢印コード」と呼ばれる）
string Process(string? input)
{
    if (input != null)
    {
        if (input.Length > 0)
        {
            if (input.StartsWith("A"))
            {
                return input.ToUpper();
            }
        }
    }
    return "";
}

// 推奨: ガード節で早期リターン
string ProcessGuard(string? input)
{
    if (input is null) return "";           // null ガード
    if (input.Length == 0) return "";       // 空文字ガード
    if (!input.StartsWith("A")) return "";  // 条件ガード

    return input.ToUpper();  // 正常処理はここ 1 箇所
}
```

ガード節を使うことで、メソッドの「正常系」が末尾に 1 か所まとまり、読みやすくなります。

## 使い分けの基準

| 状況 | 推奨 |
|------|------|
| 条件が 1〜2 個 | `if / else` |
| 多分岐・型による分岐 | `switch 式` |
| コレクションの走査 | `foreach`（インデックス不要時）|
| インデックスが必要 | `for` |
| 回数が不定 | `while` |
| 前提条件のチェック | ガード節（早期リターン）|

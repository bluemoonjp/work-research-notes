## class と struct の違い

C# の型は大きく「参照型」と「値型」に分かれます。`class` は参照型、`struct` は値型です。

| 特性 | class | struct |
|---|---|---|
| メモリ配置 | ヒープ | スタック（原則） |
| 代入時の動作 | 参照のコピー | 値のコピー |
| null 許容 | 可 | 不可（`Nullable<T>` を除く） |
| 継承 | 可 | 不可（インターフェース実装は可） |
| デフォルトコンストラクタ | 暗黙に生成 | 暗黙に生成（C# 10+: 明示定義も可） |

### コピーセマンティクスの違い

`class` は代入しても同じオブジェクトを参照するため、一方を変更するともう一方にも影響します。`struct` は代入時に値がコピーされるため、独立した別のコピーになります。

```csharp
// class: 参照のコピー
var a = new PointClass(1, 2);
var b = a;       // 同じオブジェクトを参照
b.X = 99;
Console.WriteLine(a.X); // 99（a も変わる）

// struct: 値のコピー
var p = new PointStruct(1, 2);
var q = p;       // 独立したコピー
q.X = 99;
Console.WriteLine(p.X); // 1（p は変わらない）
```

### 使い分けの基準

**struct を選ぶケース:**
- 小さなデータ（16 バイト以下が目安）
- 不変（イミュータブル）なデータ
- 大量にアロケーションされる数値・座標・色などの値オブジェクト
- 継承が不要な場合

**class を選ぶケース:**
- 大きなデータ
- 継承・ポリモーフィズムが必要
- null を表現したい
- ライフサイクル管理（`IDisposable` など）が必要

> **よくある落とし穴:** `struct` を変更可能（ミュータブル）にすると、コピーの意識が薄れて予期しない動作を引き起こすことがあります。`struct` は原則イミュータブルに設計しましょう。

## record と record struct（C# 9+/10+）

`record` は「値等値性（Value Equality）」と「非破壊的変更（with 式）」を持つ参照型です。`record struct`（C# 10+）はそれを値型で実現します。

```csharp
// record: 参照型・イミュータブル
record Person(string Name, int Age);

var alice = new Person("Alice", 30);
var bob = alice with { Name = "Bob" }; // 新しいインスタンスを作成
Console.WriteLine(alice == bob);       // False（値で比較）
```

`class` との主な違い:

| 特性 | class | record |
|---|---|---|
| 等値比較 | 参照の同一性 | 値の等値性 |
| `ToString()` | 型名 | プロパティ一覧 |
| 非破壊的変更 | 手動実装が必要 | `with` 式で簡単に |
| 不変性 | 手動で設計 | `init` プロパティがデフォルト |

**使い分けの基準:**
- DTO（Data Transfer Object）や設定データなど、不変データの表現に `record` を使います
- 状態を持つビジネスロジック（エンティティ）には `class` を使います

## sealed クラス

`sealed` を付けると、そのクラスをサブクラス化できなくなります。

```csharp
sealed class Singleton
{
    // このクラスを継承しようとするとコンパイルエラー
}
```

**用途:**
- シングルトンや不変の値オブジェクトなど、継承を意図しない型に付けます
- JIT コンパイラが仮想ディスパッチをインライン化できるため、わずかにパフォーマンスが向上することがあります

## アクセス修飾子

| 修飾子 | アクセス範囲 |
|---|---|
| `public` | どこからでもアクセス可 |
| `private` | 同じクラス・構造体内のみ（省略時のデフォルト） |
| `protected` | 同クラスおよび派生クラス |
| `internal` | 同じアセンブリ（プロジェクト）内 |
| `protected internal` | 同アセンブリ内、または派生クラス |
| `private protected` | 同アセンブリ内の派生クラスのみ |

**よくある落とし穴:** フィールドを `public` にすると、外部から自由に変更でき不変性が保てません。フィールドは `private` にして、必要に応じてプロパティでアクセスを制御しましょう。

## 静的メンバーと静的クラス

`static` メンバーはインスタンスではなく型そのものに属します。

```csharp
class Counter
{
    private static int _count = 0;     // 全インスタンスで共有
    public static int Count => _count; // 静的プロパティ

    public Counter() => _count++;
}
```

`static class` はインスタンス化できないクラスです。ユーティリティメソッドのまとまりに使います。

```csharp
static class MathUtils
{
    public static int Square(int x) => x * x;
}
```

## partial クラス

`partial` キーワードにより、1 つのクラス定義を複数のファイルに分割できます。コンパイル時に統合されます。

```csharp
// File1.cs
partial class MyClass
{
    public void MethodA() { }
}

// File2.cs
partial class MyClass
{
    public void MethodB() { }
}
```

**主な用途:** コードジェネレーター（WinForms の Designer.cs、Entity Framework のモデルなど）が生成したコードと、手書きのコードを別ファイルに分けるときに使います。

## オブジェクト初期化子

コンストラクタ呼び出し後に `{ }` でプロパティをまとめて初期化できます。

```csharp
var person = new Person
{
    Name = "Alice",
    Age = 30,
};
```

コレクション初期化子と組み合わせることもできます。

```csharp
var people = new List<Person>
{
    new Person { Name = "Alice", Age = 30 },
    new Person { Name = "Bob",   Age = 25 },
};
```

**注意:** オブジェクト初期化子はコンストラクタで保証できる不変性を回避するため、`init` アクセサ（次の記事を参照）と組み合わせて使うのが安全です。

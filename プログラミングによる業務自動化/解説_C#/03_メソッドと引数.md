# 3: メソッドと引数

## この記事で学ぶこと

- メソッドの基本構文（アクセス修飾子・戻り値・引数）
- オーバーロード
- デフォルト引数と名前付き引数
- `out` / `ref` / `in` パラメータの使い分け
- `params` キーワード（可変長引数）
- ローカル関数

## メソッドの基本構文

```csharp
// アクセス修飾子 戻り値の型 メソッド名(引数リスト)
public static int Add(int a, int b)
{
    return a + b;
}

// 式形式メンバー（C# 6+）: 本体が 1 式の場合に使える
public static int Add(int a, int b) => a + b;
```

### アクセス修飾子

| 修飾子 | 可視範囲 |
|--------|---------|
| `public` | どこからでもアクセス可能 |
| `private` | 同じクラス内のみ（クラスメンバーのデフォルト） |
| `protected` | 同じクラスおよび派生クラス |
| `internal` | 同じアセンブリ内 |
| `protected internal` | 同じアセンブリ内、または派生クラス |

## オーバーロード

引数の型や数が異なる同名メソッドを複数定義できます。戻り値の型だけが異なるオーバーロードは不可です。

```csharp
public static int Add(int a, int b) => a + b;
public static double Add(double a, double b) => a + b;
public static string Add(string a, string b) => a + b;  // 文字列連結

// 呼び出し側: コンパイラが引数の型から自動選択
Add(1, 2);         // → int 版
Add(1.0, 2.0);     // → double 版
Add("a", "b");     // → string 版
```

## デフォルト引数

引数にデフォルト値を指定すると、呼び出し時に省略できます。

```csharp
public static void Log(string message, string level = "INFO", bool timestamp = true)
{
    string prefix = timestamp ? $"[{DateTime.Now:HH:mm:ss}] " : "";
    Console.WriteLine($"{prefix}[{level}] {message}");
}

// 呼び出し例
Log("開始");                              // level="INFO", timestamp=true
Log("警告メッセージ", "WARN");            // timestamp=true
Log("タイムスタンプなし", timestamp: false); // 名前付き引数で特定の引数だけ指定
```

### よくある落とし穴

- デフォルト引数の値はコンパイル時定数でなければなりません（`const` と同様の制約）。
- デフォルト引数はオーバーロードと組み合わせると曖昧になる場合があります。呼び出し側が期待する動作を確認してください。

## 名前付き引数

引数を名前で指定することで、順序を入れ替えたり、読みやすさを上げたりできます。

```csharp
// 引数が多いメソッドで意図を明確にする
var rect = CreateRectangle(width: 100, height: 50, x: 10, y: 20);

// 名前付き引数と位置引数の混用（名前付き引数は位置引数の後に置く）
Log("メッセージ", timestamp: false);
```

## `out` / `ref` / `in` パラメータ

通常、C# のメソッド引数は**値渡し**（コピー）です。`out`、`ref`、`in` はこの挙動を変えます。

### `out`: メソッドが値を出力する

呼び出し側がメソッドに値を渡す必要はなく、メソッド側で必ず値を設定します。

```csharp
// int.TryParse は out を使う代表例
if (int.TryParse("42", out int result))
{
    Console.WriteLine(result); // → 42
}

// 自分で out パラメータを持つメソッドを定義する
public static bool TryDivide(int dividend, int divisor, out int quotient)
{
    if (divisor == 0)
    {
        quotient = 0;        // out は必ず代入する
        return false;
    }
    quotient = dividend / divisor;
    return true;
}
```

### `ref`: 参照渡し（双方向）

呼び出し側の変数そのものを渡します。メソッド内での変更が呼び出し元に反映されます。

```csharp
public static void Increment(ref int value)
{
    value++;  // 呼び出し元の変数に直接書き込む
}

int n = 10;
Increment(ref n);
Console.WriteLine(n); // → 11
```

`ref` は主にパフォーマンス最適化（大きな struct のコピーを避ける）で使います。通常のコードでは `out` か戻り値で代替できる場合が多いです。

### `in`: 参照渡しだが読み取り専用

大きな `struct` を参照渡しで渡しつつ、メソッド内での変更を防ぎます。

```csharp
public static double ComputeDistance(in Point p1, in Point p2)
{
    // p1.X = 0; // コンパイルエラー: in パラメータは読み取り専用
    return Math.Sqrt(Math.Pow(p2.X - p1.X, 2) + Math.Pow(p2.Y - p1.Y, 2));
}
```

### 使い分けの基準

| パラメータ | 方向 | 用途 |
|-----------|------|------|
| （なし） | 入力 | 通常の引数渡し |
| `out` | 出力 | メソッドが複数の値を返す、Try パターン |
| `ref` | 双方向 | パフォーマンス最適化（大きな struct）、値型の変更 |
| `in` | 読み取り専用入力 | 大きな struct をコピーせずに渡す |

## `params`（可変長引数）

引数の個数が不定の場合に使います。

```csharp
public static int Sum(params int[] numbers)
{
    return numbers.Sum();  // LINQ の Sum()
}

// 呼び出し: 個数を気にせず渡せる
Console.WriteLine(Sum(1, 2, 3));         // → 6
Console.WriteLine(Sum(10, 20, 30, 40)); // → 100

// 配列を渡すこともできる
int[] data = { 5, 10, 15 };
Console.WriteLine(Sum(data));            // → 30
```

### よくある落とし穴

- `params` はメソッドの最後の引数にしか使えません。
- `params` と既存のオーバーロードが曖昧にならないよう注意してください。

## ローカル関数

メソッドの中にメソッドを定義できます。外側のメソッドのスコープ内でしか使わないヘルパー処理をカプセル化するのに便利です。

```csharp
public static IEnumerable<int> GetEvenNumbers(int[] input)
{
    // ローカル関数: このメソッドの中でのみ使える
    bool IsEven(int n) => n % 2 == 0;

    foreach (var n in input)
    {
        if (IsEven(n))
            yield return n;
    }
}
```

### ローカル関数 vs ラムダ式

| 観点 | ローカル関数 | ラムダ式（`Func`/`Action`） |
|------|------------|--------------------------|
| 再帰 | 可能 | 難しい（変数経由で自己参照が必要） |
| デバッグ時の名前 | あり（スタックトレースに出る） | 匿名（デバッグしにくい場合がある） |
| `yield return` | 使える | 使えない |
| 型推論 | コンパイラが行う | `Func`/`Action` の型を明示する場合がある |

**複雑なロジックや再帰が必要な場合はローカル関数**、**LINQ や `Action`/`Func` に渡す短い処理はラムダ式**が一般的です。

## デリゲートとは

デリゲートは「メソッドへの型安全な参照」です。C 言語の関数ポインタに相当しますが、型チェックが厳密で安全に使えます。

```csharp
// デリゲート型の定義（int を受け取り string を返す）
delegate string Formatter(int value);

// デリゲート変数に関数を代入
Formatter fmt = n => $"値: {n}";

// デリゲートを呼び出す
Console.WriteLine(fmt(42)); // "値: 42"
```

独自デリゲート型を定義する場面は現在ほとんどありません。組み込みの `Func<>` / `Action<>` / `Predicate<>` で大半のケースをカバーできます。

---

## Func<> / Action<> / Predicate<>

| 型 | シグネチャ | 用途 |
|---|---|---|
| `Func<TResult>` | 引数なし、戻り値 `TResult` | 値を生成する関数 |
| `Func<T, TResult>` | 引数 1 つ、戻り値 `TResult` | 変換・計算 |
| `Func<T1, T2, TResult>` | 引数 2 つ… 最大 16 引数まで | 同上 |
| `Action` | 引数なし、戻り値なし | 副作用のみの処理 |
| `Action<T>` | 引数 1 つ、戻り値なし | 同上 |
| `Predicate<T>` | 引数 1 つ、戻り値 `bool` | 条件チェック（`Func<T, bool>` と等価） |

```csharp
Func<int, int, int> add   = (a, b) => a + b;
Action<string>      print = msg => Console.WriteLine(msg);
Predicate<int>      isEven = n => n % 2 == 0;

Console.WriteLine(add(3, 4));   // 7
print("Hello");
Console.WriteLine(isEven(6));   // True
```

LINQ のメソッドチェーン（`Where`, `Select` など）では `Func<>` と `Predicate<>` が多用されます。

## ラムダ式

ラムダ式は匿名関数をコンパクトに書く構文です。

### 式形式（Expression Lambda）

右辺が 1 つの式だけのとき、`{}` と `return` を省略できます。

```csharp
Func<int, int> square = x => x * x;
Func<int, int, int> add = (a, b) => a + b;
```

### ブロック形式（Statement Lambda）

複数の処理が必要な場合は `{}` を使います。

```csharp
Func<int, string> classify = n =>
{
    if (n < 0) return "負";
    if (n == 0) return "ゼロ";
    return "正";
};
```

**使い分けの基準:** 1 式で収まるなら式形式、複数処理が必要ならブロック形式。ブロック形式が長くなる場合は、名前付きメソッドとして切り出すとテストしやすくなります。

## マルチキャストデリゲート

デリゲートは `+=` で複数のメソッドを連結（チェーン）できます。呼び出すと登録順に全メソッドが実行されます。

```csharp
Action<string> logger  = msg => Console.WriteLine($"[LOG] {msg}");
Action<string> auditor = msg => Console.WriteLine($"[AUDIT] {msg}");

Action<string> combined = logger + auditor;
combined("ログイン");
// [LOG] ログイン
// [AUDIT] ログイン

combined -= logger; // 登録解除
combined("ログアウト"); // [AUDIT] ログアウト のみ
```

> **よくある落とし穴:** 戻り値がある `Func<>` のマルチキャストでは、最後に登録されたメソッドの戻り値だけが返ります。戻り値が必要な場合はマルチキャストを避け、明示的にリストで管理しましょう。

## イベント（event キーワード）

`event` はデリゲートを「パブリッシャー/サブスクライバー」パターンで安全に公開するためのキーワードです。クラス外部からは `+=` / `-=` のみ可能で、直接呼び出しや代入（`=`）は禁止されます。

```csharp
class Button
{
    // EventHandler は標準のイベント用デリゲート型（object sender, EventArgs e）
    public event EventHandler? Clicked;

    public void Click()
    {
        // イベント発火（null チェックが必要）
        Clicked?.Invoke(this, EventArgs.Empty);
    }
}

var btn = new Button();

// サブスクライバーの登録
btn.Clicked += (sender, e) => Console.WriteLine("クリックされました");
btn.Clicked += (sender, e) => Console.WriteLine("もう一人も反応");

btn.Click(); // 両方のハンドラが呼ばれる
```

カスタムデータをイベントで渡したい場合は `EventArgs` のサブクラスを定義します。

```csharp
class ClickEventArgs : EventArgs
{
    public int ClickCount { get; }
    public ClickEventArgs(int count) => ClickCount = count;
}

class AdvancedButton
{
    public event EventHandler<ClickEventArgs>? Clicked;
    private int _clickCount;

    public void Click()
    {
        _clickCount++;
        Clicked?.Invoke(this, new ClickEventArgs(_clickCount));
    }
}
```

## クロージャ（変数キャプチャ）

ラムダ式は自分が定義されたスコープの変数を「キャプチャ（参照）」できます。これをクロージャといいます。

```csharp
int multiplier = 3;
Func<int, int> triple = x => x * multiplier; // multiplier をキャプチャ

Console.WriteLine(triple(5)); // 15

multiplier = 10; // キャプチャした変数が変わると結果も変わる
Console.WriteLine(triple(5)); // 50（コピーではなく参照のため）
```

### よくある落とし穴: ループ変数のキャプチャ

`for` ループ内でラムダ式を生成すると、ループ変数 `i` を参照キャプチャします。ループ終了後に全てのラムダを実行すると、全てが最終値を参照します。

```csharp
var actions = new List<Action>();

for (int i = 0; i < 3; i++)
{
    actions.Add(() => Console.WriteLine(i)); // i を参照キャプチャ
}

// ループ終了後: i は 3 になっているので、全部 3 が出力される
foreach (var action in actions)
    action(); // 3, 3, 3（期待値: 0, 1, 2）
```

**修正方法:** ループ内でローカル変数にコピーし、そちらをキャプチャします。

```csharp
for (int i = 0; i < 3; i++)
{
    int captured = i; // ループごとに独立したコピー
    actions.Add(() => Console.WriteLine(captured));
}
// 0, 1, 2（期待通り）
```

`foreach` ループでは C# 5 以降、ループ変数が各イテレーションで独立したコピーになるため、この問題は発生しません。`for` ループを使う場合にのみ注意が必要です。

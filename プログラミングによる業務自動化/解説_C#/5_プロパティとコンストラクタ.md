## プロパティとフィールドの違い

**フィールド**はクラスが直接持つ変数です。**プロパティ**はフィールドへのアクセスをラップするメンバーで、`get` / `set` アクセサにロジックを挟めます。

```csharp
class Temperature
{
    // フィールド（private）
    private double _celsius;

    // プロパティ（public）: バリデーション付き
    public double Celsius
    {
        get => _celsius;
        set
        {
            if (value < -273.15)
                throw new ArgumentOutOfRangeException(nameof(value));
            _celsius = value;
        }
    }

    // 計算プロパティ（バッキングフィールドなし）
    public double Fahrenheit => _celsius * 9 / 5 + 32;
}
```

**使い分けの基準:**
- クラス内部でのみ使うデータ → `private` フィールド
- 外部に公開し、バリデーションや変換が必要 → プロパティ
- 単純に読み書きするだけ → 自動実装プロパティ

> **よくある落とし穴:** フィールドを `public` で直接公開すると、後からバリデーションを追加できません。最初からプロパティとして公開しておくと変更に強い設計になります。

## 自動実装プロパティ

`get` / `set` に処理が不要な場合、コンパイラが自動的にバッキングフィールドを生成します。

```csharp
class Person
{
    public string Name { get; set; } = string.Empty; // デフォルト値も設定可
    public int Age { get; set; }

    // 読み取り専用（set がない）
    public DateTime CreatedAt { get; } = DateTime.Now;
}
```

`{ get; }` のみの場合、コンストラクタ内でのみ代入できます（それ以外は読み取り専用）。

## init アクセサ（C# 9+）

`init` は「オブジェクト初期化子のときだけ書き込める」アクセサです。コンストラクタ後の変更を防ぎつつ、オブジェクト初期化子の便利さを維持できます。

```csharp
class Config
{
    public string Host { get; init; } = "localhost";
    public int Port { get; init; } = 8080;
}

// 初期化子で設定 OK
var config = new Config { Host = "example.com", Port = 443 };

// 初期化後の変更はコンパイルエラー
// config.Host = "other.com"; // エラー
```

`record` は内部的に `init` を使っているため、`with` 式で非破壊的変更が可能です。

## 式形式プロパティ

`=>` を使ってプロパティを 1 行で書けます。計算プロパティや単純な委譲に適しています。

```csharp
class Circle
{
    public double Radius { get; }

    // 式形式プロパティ（ゲッターのみ）
    public double Area => Math.PI * Radius * Radius;
    public double Circumference => 2 * Math.PI * Radius;

    public Circle(double radius) => Radius = radius;
}
```

## コンストラクタのオーバーロード

C# では同じクラスに複数のコンストラクタを定義できます（引数の型・数が異なる必要があります）。

```csharp
class Rectangle
{
    public double Width { get; }
    public double Height { get; }

    // 引数 2 つのコンストラクタ
    public Rectangle(double width, double height)
    {
        Width = width;
        Height = height;
    }

    // 正方形用: 辺の長さだけ指定（引数 1 つ）
    public Rectangle(double side) : this(side, side)  // コンストラクタ委譲
    {
    }
}
```

## this(...) によるコンストラクタ委譲

`: this(...)` を使うと、別のコンストラクタを呼び出してからメンバー初期化のロジックを共通化できます。重複を避けるために有効です。

```csharp
class Server
{
    public string Host { get; }
    public int Port { get; }
    public string Protocol { get; }

    // 共通のコンストラクタ（最も多い引数）
    public Server(string host, int port, string protocol)
    {
        Host = host;
        Port = port;
        Protocol = protocol;
    }

    // デフォルトポートを指定するショートカット
    public Server(string host, string protocol)
        : this(host, protocol == "https" ? 443 : 80, protocol) { }

    // さらに省略形
    public Server(string host)
        : this(host, "http") { }
}
```

> **よくある落とし穴:** `: this(...)` を使わずにコンストラクタ間でロジックをコピーすると、片方を修正し忘れたときにバグの温床になります。共通処理は必ず委譲でまとめましょう。

## プライマリコンストラクタ（C# 12+）

C# 12 でクラス・構造体にもプライマリコンストラクタが導入されました（record では以前から使用可）。クラス宣言の引数リストがコンストラクタ引数になります。

```csharp
// 従来の書き方
class Point
{
    public double X { get; }
    public double Y { get; }

    public Point(double x, double y) { X = x; Y = y; }
}

// プライマリコンストラクタ（C# 12+）
class PointV2(double x, double y)
{
    public double X { get; } = x;
    public double Y { get; } = y;

    public double DistanceTo(PointV2 other)
        => Math.Sqrt(Math.Pow(X - other.X, 2) + Math.Pow(Y - other.Y, 2));
}
```

プライマリコンストラクタのパラメータはクラス全体のスコープで使えます（フィールドへの代入なしに直接参照可）。

> **よくある落とし穴:** プライマリコンストラクタのパラメータは変更可能（`set` 可能）です。`record` のように自動的にイミュータブルにはならないため、意図的に `{ get; } = param;` とプロパティに代入する必要があります。

## デストラクタとファイナライザ

C# では `~ClassName()` でファイナライザを定義できます。ガベージコレクタがオブジェクトを回収するときに呼ばれますが、**呼ばれるタイミングは保証されません**。

```csharp
class ResourceHolder
{
    ~ResourceHolder()
    {
        // GC 回収時に呼ばれるが、タイミング不定
    }
}
```

**ほとんどのケースではファイナライザを使わないでください。** アンマネージドリソースの解放には `IDisposable` パターンと `using` ステートメントを使うのが C# の標準的なやり方です（別記事参照）。

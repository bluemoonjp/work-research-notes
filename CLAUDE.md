# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## リポジトリ概要

仕事で使える技術的な調査結果を記録するナレッジベース。AI が調査した内容を蓄積することを主目的とする。

## コンテンツルール

- **1ページ1テーマ**: 各 Markdown ファイルは単一のテーマ・トピックを扱う
- **出典リンク必須**: 主要な情報源は必ずリンクを残す
- **組織情報の排除**: 所属組織の固有名詞・内部情報はリポジトリに含めない

## ファイル・ディレクトリ命名規則

- README.md・CLAUDE.md などの標準ファイルを除き、ファイル名・ディレクトリ名は日本語を使用する
- コードサンプルは記事と同じディレクトリに置き、ファイル名で言語を区別する（例: `ブラウザ操作_CSHARP.cs`）

## 調査規約

- **調査**: Codex と Claude サブエージェントで並行して実施する
- **レビュー**: Codex と Claude サブエージェントがそれぞれ独立してレビューを行う
- **再調査**: レビュー結果に基づき必要と判断した場合は再調査を行う

## Git 規約

- ファイルに変更を加えたら、都度 `git commit` する

## 言語規約

- **基本想定言語**: C# および VBA
- コードサンプルを作成する際は、特に指定がない限り C# または VBA を使用する

## ディレクトリ構成パターン

大テーマ → 中テーマ（任意） → 小テーマ（任意） → 記事・コードサンプル

```
<大テーマ>/
  <記事>.md
  <中テーマ>/
    <記事>.md
    <小テーマ>/
      <記事>.md
      <コードサンプル>.<拡張子>
```

## プログラミングによる業務自動化

### ディレクトリ命名プレフィックス

| プレフィックス | 用途 | 例 |
|---|---|---|
| `調査_` | 特定テーマの実装方法・手法の調査 | `調査_認証情報の扱い方` |
| `比較_` | 複数言語・手法の比較検討 | `比較_Excel操作(VBA・C#)` |
| `サンプルコード_` | フレームワーク・プラットフォーム別のサンプル集 | `サンプルコード_.NET` |

- `比較_` ディレクトリは末尾に `(<言語1>・<言語2>)` を付ける

### コードファイル命名

```
<連番>_<説明>_<言語識別子>.<拡張子>
```

- 連番は 1 始まりの整数（ゼロ埋め不要）
- 言語識別子: `CSHARP`（.cs）、`VBA`（.bas）
- 比較ディレクトリ内では同じ連番を複数言語で共有する（対になる）

### Markdown ファイル

| ファイル名 | 要否 | 内容 |
|---|---|---|
| `README.md` | 各テーマディレクトリに必須 | テーマの目的・方針・コード一覧 |
| `言語別解説.md` | 複数言語が存在する場合 | 各言語ごとのコードの解説 |
| `<テーマ固有>.md` | 任意 | 比較表・特徴まとめなど |
